# WeKnora 快速部署指南

## 🚀 5分钟快速部署

### 前置条件
- ✅ Docker 和 Docker Compose 已安装
- ✅ 至少 8GB 内存和 50GB 磁盘空间

### 步骤1：克隆项目
```bash
git clone https://github.com/Tencent/WeKnora.git
cd WeKnora
```

### 步骤2：配置环境变量
```bash
# 创建.env文件（如果不存在）
cp .env.example .env  # 如果存在模板文件

# 编辑.env文件，至少配置以下必需项：
# - DB_USER, DB_PASSWORD, DB_NAME
# - REDIS_PASSWORD
# - JWT_SECRET
# - TENANT_AES_KEY
```

### 步骤3：一键启动
```bash
./scripts/start_all.sh
```

### 步骤4：访问服务
- 🌐 Web界面：http://localhost
- 🔌 API接口：http://localhost:8080

---

## 📋 最小化配置示例

创建 `.env` 文件，填入以下最小配置：

```bash
# ========== 数据库配置 ==========
DB_DRIVER=postgres
DB_USER=weknora
DB_PASSWORD=weknora123456
DB_NAME=weknora

# ========== Redis配置 ==========
REDIS_PASSWORD=redis123456

# ========== 存储配置 ==========
STORAGE_TYPE=local
LOCAL_STORAGE_BASE_DIR=/data/files

# ========== 安全配置 ==========
JWT_SECRET=your_jwt_secret_key_change_this_in_production
TENANT_AES_KEY=your_aes_key_change_this_in_production

# ========== Ollama配置 ==========
OLLAMA_BASE_URL=http://host.docker.internal:11434

# ========== 端口配置 ==========
APP_PORT=8080
FRONTEND_PORT=80
```


## 🔧 功能开关

### 启用知识图谱（Neo4j）
```bash
# 1. 在.env中添加配置
ENABLE_GRAPH_RAG=true
NEO4J_ENABLE=true
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=neo4j123456

# 2. 启动Neo4j服务
docker-compose --profile neo4j up -d
```

### 启用对象存储（MinIO）
```bash
# 1. 在.env中配置
STORAGE_TYPE=minio
MINIO_ACCESS_KEY_ID=minioadmin
MINIO_SECRET_ACCESS_KEY=minioadmin
MINIO_BUCKET_NAME=weknora-files

# 2. 启动MinIO服务
docker-compose --profile minio up -d

# 3. 访问MinIO控制台
# http://localhost:9001
# 用户名：minioadmin
# 密码：minioadmin
```

### 启用链路追踪（Jaeger）
```bash
# 启动Jaeger服务
docker-compose --profile jaeger up -d

# 访问Jaeger UI
# http://localhost:16686
```

## 📦 模型配置

### 使用Ollama本地模型

1. **安装Ollama**
   ```bash
   # Linux/macOS
   curl -fsSL https://ollama.com/install.sh | sh
   
   # 或使用脚本自动安装
   ./scripts/start_all.sh --ollama
   ```

2. **下载模型**
   ```bash
   # LLM模型
   ollama pull qwen2.5:7b
   
   # Embedding模型
   ollama pull bge-large-zh-v1.5
   
   # Rerank模型（可选）
   ollama pull bge-reranker-large
   ```

3. **配置.env**
   ```bash
   OLLAMA_BASE_URL=http://host.docker.internal:11434
   INIT_LLM_MODEL_NAME=qwen2.5:7b
   INIT_LLM_MODEL_BASE_URL=http://host.docker.internal:11434
   INIT_EMBEDDING_MODEL_NAME=bge-large-zh-v1.5
   INIT_EMBEDDING_MODEL_BASE_URL=http://host.docker.internal:11434
   INIT_EMBEDDING_MODEL_DIMENSION=1024
   ```

### 使用远程API模型

```bash
# 在.env中配置
INIT_LLM_MODEL_NAME=qwen-plus
INIT_LLM_MODEL_BASE_URL=https://api.example.com/v1
INIT_LLM_MODEL_API_KEY=your_api_key

INIT_EMBEDDING_MODEL_NAME=text-embedding-ada-002
INIT_EMBEDDING_MODEL_BASE_URL=https://api.openai.com/v1
INIT_EMBEDDING_MODEL_API_KEY=your_api_key
INIT_EMBEDDING_MODEL_DIMENSION=1536
```

## 🐛 常见问题快速解决

### 问题1：端口被占用
```bash
# 检查端口占用
lsof -i :8080  # Linux/macOS
netstat -ano | findstr :8080  # Windows

# 修改.env中的端口配置
APP_PORT=8081
FRONTEND_PORT=81
```

### 问题2：容器启动失败
```bash
# 查看详细日志
docker compose logs app
docker compose logs postgres

# 检查环境变量
docker compose config

# 重启服务
docker compose down
docker compose up -d
```

### 问题3：数据库连接失败
```bash
# 检查数据库是否运行
docker compose ps postgres

# 测试数据库连接
docker compose exec postgres psql -U weknora -d weknora

# 检查.env中的数据库配置
cat .env | grep DB_
```

### 问题4：前端无法访问
```bash
# 检查前端容器
docker compose ps frontend

# 查看前端日志
docker compose logs frontend

# 检查Nginx配置
docker compose exec frontend cat /etc/nginx/nginx.conf
```

## 🔐 安全建议

### 生产环境必须修改
```bash
# 1. 修改所有默认密码
DB_PASSWORD=strong_password_here
REDIS_PASSWORD=strong_password_here
NEO4J_PASSWORD=strong_password_here

# 2. 生成强密钥
JWT_SECRET=$(openssl rand -hex 32)
TENANT_AES_KEY=$(openssl rand -hex 32)

# 3. 禁用注册（可选）
DISABLE_REGISTRATION=true

# 4. 配置HTTPS（使用Nginx反向代理）
```
