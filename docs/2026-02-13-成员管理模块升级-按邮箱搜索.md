# 成员管理模块升级：按邮箱/姓名搜索

> **文档创建**：2026-02-13  
> **功能**：知识库成员管理增加按 email、username、姓名（cas_real_name）的后端搜索能力

---

## 一、升级概述

| 项目 | 说明 |
|------|------|
| **升级前** | 知识库成员管理仅支持分页，无法按邮箱或姓名搜索 |
| **升级后** | 支持按关键词搜索成员（匹配 email、username、cas_real_name），与分页配合使用 |
| **实现方式** | 后端搜索（非前端过滤），支持跨页全局检索 |

---

## 二、修改点汇总

| 层级 | 文件 | 修改内容 |
|------|------|----------|
| Handler | `internal/handler/knowledgebase.go` | 从 query 读取 `keyword`，传入 Service |
| Service | `internal/application/service/shared_kb.go` | `ListKnowledgeBaseMembers` 增加 `keyword string` 参数 |
| Interface | `internal/types/interfaces/shared_kb.go` | `ListKnowledgeBaseMembers`、`ListMembersByKB` 增加 `keyword` 参数 |
| Repository | `internal/application/repository/kb_member.go` | `ListMembersByKB` 增加 `keyword`，keyword 非空时 JOIN users 并按 email/username/cas_real_name LIKE 过滤 |
| API | `frontend/src/api/knowledge-base/index.ts` | `listKnowledgeBaseMembers` 增加可选参数 `keyword?: string` |
| 组件 | `frontend/src/views/knowledge/settings/KnowledgeBaseMembers.vue` | 增加搜索框、debounce 300ms、搜索时重置 page 为 1 |
| i18n | `frontend/src/i18n/locales/zh-CN.ts`、`en-US.ts` | 新增 `searchPlaceholder` 文案 |

## 三、API 变更

### 3.1 列出知识库成员

**接口**：`GET /api/v1/knowledge-bases/:id/members`

**新增 Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `keyword` | string | 否 | 搜索关键词，匹配成员 email、username、cas_real_name（LIKE %keyword%） |

**原有参数**（不变）：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `page` | int | 否 | 页码，默认 1 |
| `page_size` | int | 否 | 每页数量，默认 10，最大 100 |

**示例**：

```
GET /api/v1/knowledge-bases/{kbId}/members?keyword=zhang&page=1&page_size=20
```

## 四、后端实现要点

### 4.1 搜索逻辑

- **keyword 为空**：保持原有逻辑，不做过滤
- **keyword 非空**：`JOIN users ON users.id = knowledge_base_members.user_id`，使用  
  `WHERE users.email LIKE ? OR users.username LIKE ? OR users.cas_real_name LIKE ?`（参数化，防 SQL 注入）

### 4.2 Count 与列表一致性

- 统计总数（Count）与列表查询使用相同的 WHERE 条件，确保分页 total 正确

### 4.3 表与字段

- `knowledge_base_members`：成员表，`user_id` 关联 `users.id`
- `users`：用户表，包含 `email`、`username`、`cas_real_name`

---

## 五、前端实现要点

### 5.1 搜索框

- 位置：成员列表上方，`KnowledgeBaseMembers.vue` 内
- 输入 debounce 300ms，避免频繁请求
- 输入变更时 `currentPage` 重置为 1

### 5.2 API 调用

```typescript
listKnowledgeBaseMembers(kbId.value, {
  keyword: searchKeyword.value.trim() || undefined,
  page: currentPage.value,
  page_size: pageSize.value
})
```

### 5.3 国际化

| 语言 | 文案 |
|------|------|
| zh-CN | 按邮箱或姓名搜索 |
| en-US | Search by email or name |

## 六、影响范围
