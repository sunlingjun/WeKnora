## 二、版本 2：统一颜色升级（今日新增内容）

### 2.1 今日新增文件清单

| 文件 | 修改内容 | 说明 |
|------|----------|------|
| `frontend/src/main.ts` | 应用启动时从 localStorage 读取主题并设置 `theme-mode` | 主题持久化与启动恢复 |
| `frontend/src/views/settings/GeneralSettings.vue` | 新增外观主题选择行（t-select：light/dark/system），监听系统偏好变化 | 用户界面入口 |
| `frontend/src/i18n/locales/zh-CN.ts` | 新增 `general.appearance` 下 theme、themeDescription、themeLight、themeDark、themeSystem、themeSaved | 中文文案 |
| `frontend/src/i18n/locales/en-US.ts` | 同上 | 英文文案 |
| `frontend/src/i18n/locales/ru-RU.ts` | 同上 | 俄语文案 |
| `frontend/src/i18n/locales/ko-KR.ts` | 同上 | 韩语文案 |

### 2.2 与已有颜色统一化改造的关系

- **已有基础**：`frontend/src/assets/theme/theme.css` 已定义 `:root[theme-mode="light"]` 与 `:root[theme-mode="dark"]` 变量
- **今日增强**：增加用户可切换主题的 UI 入口与持久化机制
- **依赖关系**：今日升级依赖「颜色统一化改造」中已完成的 CSS 变量体系

### 2.3 贡献建议

- **分支名**：`feature/color-unify-tdesign-vars`（与已有颜色统一化改造合并）
- **提交信息**：`feat(frontend): add theme switcher in general settings`
- **PR 描述**：说明新增主题切换功能，依赖已有的 TDesign CSS 变量体系，支持浅色/深色/跟随系统三种模式

## 四、版本 3：成员管理模块升级（独立版本，可选）

### 4.1 文件清单

| 层级 | 文件 | 修改内容 |
|------|------|----------|
| Handler | `internal/handler/knowledgebase.go` | 从 query 读取 `keyword`，传入 Service |
| Service | `internal/application/service/shared_kb.go` | `ListKnowledgeBaseMembers` 增加 `keyword string` 参数 |
| Interface | `internal/types/interfaces/shared_kb.go` | `ListKnowledgeBaseMembers`、`ListMembersByKB` 增加 `keyword` 参数 |
| Repository | `internal/application/repository/kb_member.go` | `ListMembersByKB` 增加 `keyword`，keyword 非空时 JOIN users 并按 email/username/cas_real_name LIKE 过滤 |
| API | `frontend/src/api/knowledge-base/index.ts` | `listKnowledgeBaseMembers` 增加可选参数 `keyword?: string` |
| 组件 | `frontend/src/views/knowledge/settings/KnowledgeBaseMembers.vue` | 增加搜索框、debounce 300ms、搜索时重置 page 为 1 |
| i18n | `frontend/src/i18n/locales/zh-CN.ts`、`en-US.ts` | 新增 `searchPlaceholder` 文案 |

### 4.2 贡献建议

- **分支名**：`feature/kb-member-search`（独立功能，不与其他版本合并）
- **提交信息**：`feat(backend,frontend): add keyword search for knowledge base members`
- **PR 描述**：说明新增知识库成员按邮箱/姓名搜索功能，后端支持跨页全局检索
- **优先级**：可先完成版本 1 和版本 2 的贡献，再考虑此版本

## 六、重叠文件处理

### 6.1 `frontend/src/components/Input-field.vue`

- **版本 2（颜色）**：若该文件还有其他颜色相关改动（CSS 变量替换），用 `git add -p` 只选颜色相关块
- **版本 1（SVG）**：用 `git add -p` 只选 at-icon 内联 SVG 相关块（约第 X 行到第 Y 行，具体见文件）

### 6.2 i18n 文件（zh-CN.ts、en-US.ts 等）

- **版本 2**：只 add `general.appearance` 相关键值对
- **版本 3**：只 add `searchPlaceholder` 相关键值对
- **方法**：用 `git add -p` 按块选择，或先手动拆分到不同分支再分别 add

## 七、操作步骤建议

### 7.1 场景：今日改动全部未提交（工作区脏）

```powershell
# 1. 备份当前全部修改
git stash push -u -m "all-2026-02-13-upgrades"

# 2. 确保 main 与上游一致
git checkout main
git fetch upstream
git merge upstream/main

# 3. 先做「版本 2：颜色 + 主题设置」
git checkout -b feature/color-unify-tdesign-vars
git stash pop

# 只 add 版本 2 相关文件
git add frontend/src/main.ts
git add frontend/src/views/settings/GeneralSettings.vue
git add -p frontend/src/i18n/locales/zh-CN.ts  # 只选 general.appearance
git add -p frontend/src/i18n/locales/en-US.ts  # 只选 general.appearance
git add -p frontend/src/i18n/locales/ru-RU.ts  # 只选 general.appearance
git add -p frontend/src/i18n/locales/ko-KR.ts  # 只选 general.appearance
# 若 Input-field.vue 有颜色相关改动，也用 add -p 只选颜色块
git add -p frontend/src/components/Input-field.vue  # 只选颜色相关块

git commit -m "feat(frontend): add theme switcher in general settings"
git push origin feature/color-unify-tdesign-vars

# 4. 剩余改动还在工作区；再做「版本 1：SVG 组件 + 图标适配」
git stash push -u -m "remaining-svg"
git checkout main
git checkout -b feature/svg-icon-component
git stash pop

# 只 add 版本 1 相关文件
git add -p frontend/src/components/Input-field.vue  # 只选 at-icon SVG 相关块
# 若 at-icon.svg 要删除：git rm frontend/src/assets/img/at-icon.svg

git commit -m "refactor(frontend): replace at-icon img with inline SVG using currentColor"
git push origin feature/svg-icon-component

# 5. 剩余改动再做「版本 3：成员搜索」（可选）
git stash push -u -m "remaining-member-search"
git checkout main
git checkout -b feature/kb-member-search
git stash pop

# add 版本 3 相关文件
git add internal/handler/knowledgebase.go
git add internal/application/service/shared_kb.go
git add internal/types/interfaces/shared_kb.go
git add internal/application/repository/kb_member.go
git add frontend/src/api/knowledge-base/index.ts
git add frontend/src/views/knowledge/settings/KnowledgeBaseMembers.vue
git add -p frontend/src/i18n/locales/zh-CN.ts  # 只选 searchPlaceholder
git add -p frontend/src/i18n/locales/en-US.ts  # 只选 searchPlaceholder

git commit -m "feat(backend,frontend): add keyword search for knowledge base members"
git push origin feature/kb-member-search
```

### 7.2 场景：今日改动已部分提交

- 若已提交在 main：按 [2026-02-13-贡献流程配置与待贡献版本方案](./2026-02-13-贡献流程配置与待贡献版本方案.md) 的「场景 B」处理

## 九、注意事项

1. **重叠文件**：`Input-field.vue` 和 i18n 文件需要按块（hunk）拆分，使用 `git add -p` 只选对应版本的改动
2. **依赖关系**：版本 2 的主题设置依赖已有的 CSS 变量体系，需确保 `theme.css` 已包含在版本 2 的 PR 中
3. **测试**：每个版本提交前需测试：
   - 版本 2：切换主题（浅色/深色/跟随系统）后，界面颜色正确切换
   - 版本 1：at-icon 在浅色/深色主题下颜色正确
   - 版本 3：成员搜索功能正常，分页与搜索配合正确
