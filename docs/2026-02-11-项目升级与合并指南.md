## 二、冲突分析与解决策略

### 2.1 功能冲突：组织共享 vs 直接共享

| 维度 | 上游（组织共享） | 本地（直接共享） |
|------|----------------|----------------|
| 数据表 | knowledge_base_shares | knowledge_base_members |
| 加入方式 | 加入组织自动获得 | 直接加入知识库 |
| 权限 | 组织角色 + 共享权限 | 直接成员角色 |

**方案**：双模式共存，两种机制并存，详见第三至第五节。

### 2.2 核心冲突文件（按优先级）

| 优先级 | 文件 | 解决策略 |
|--------|------|---------|
| ⭐⭐⭐⭐⭐ | Input-field.vue、menu.vue | 上游功能 + 本地样式/SVG |
| ⭐⭐⭐⭐ | theme.css、AgentSelector.vue、MentionSelector.vue | 合并变量 / 上游功能 + 本地样式 |
| ⭐⭐⭐ | AgentEditorModal、KnowledgeBase、KnowledgeBaseList 等 | 上游功能 + 本地颜色统一化 |
| ⭐⭐ | api/*、router 等 | 合并端点与逻辑 |

### 2.3 三路合并流程

```powershell
# 1. 备份
git branch backup-$(Get-Date -Format "yyyyMMdd")

# 2. 暂存本地
git stash push -u -m "本地改造"

# 3. 拉取上游
git pull origin main

# 4. 恢复
git stash pop

# 5. 解决冲突（顺序：theme.css → menu.vue → Input-field.vue → AgentSelector → 其他）
git mergetool

# 6. 验证
cd frontend && npm run lint && npm run type-check
cd .. && go build ./cmd/server
```

## 五、双模式共存（设计）

### 5.1 数据库

```sql
-- knowledge_bases 扩展
ALTER TABLE knowledge_bases ADD COLUMN IF NOT EXISTS visibility VARCHAR(32) DEFAULT 'private',
  ADD COLUMN owner_id VARCHAR(36), ADD COLUMN shared_at TIMESTAMP, ADD COLUMN member_count INT DEFAULT 0;

-- knowledge_base_members（直接共享）
CREATE TABLE IF NOT EXISTS knowledge_base_members (
  knowledge_base_id VARCHAR(36), user_id VARCHAR(36), role VARCHAR(32) DEFAULT 'viewer',
  joined_at TIMESTAMP, PRIMARY KEY (knowledge_base_id, user_id), ...
);
```

### 5.2 API 与权限

- 权限顺序：owner > 组织共享 > 直接共享
- 统一接口：`GET /api/v1/knowledge-bases`，支持 `visibility`、`organization_id`、`include_shared`
- 共享：`POST /share`（`share_mode: 'org' | 'direct'`）、`/join`、`/leave`

### 5.3 前端

- **KnowledgeBaseList**：合并「我的 + 组织共享 + 直接共享」，去重（owner 优先）
- **KBShareSettings**：组织共享（上游共享空间方案）
- **kb-permission.ts**：`canEditKB`、`canDeleteKB` 等

## 九、注意事项与常见问题

| 类型 | 说明 |
|------|------|
| 迁移 | 勿手动改迁移文件，按序执行 |
| API | 核对调用与类型定义 |
| Store | 检查 organization、cas 初始化与依赖 |
| SVG | 保留 currentColor，与上游功能整合 |

**Q：合并后 CAS 不工作？**  
A：检查 cas 相关文件是否被覆盖、认证中间件是否支持 CAS、路由守卫是否调用 casStore。

**Q：合并后出现硬编码颜色？**  
A：用 grep 查找并替换为 CSS 变量，再验证主题切换。

**Q：知识库列表重复或权限异常？**  
A：检查 `allKnowledgeBases` 去重逻辑、`shareMode`、`kb-permission.ts` 应用情况。
