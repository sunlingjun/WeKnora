# 存储配额配置说明

> **文档创建**：2025-01-27

> 📌 **文档目的**：说明存储空间配额的配置位置和修改方法

---

## 一、存储配额概述

### 1.1 配额说明

- **存储配额**：每个租户（Tenant）可使用的最大存储空间
- **单位**：字节（Bytes）
- **默认值**：**1GB** (1073741824 字节)
- **包含内容**：向量数据、原始文件、文本、索引等

### 1.2 配额值对照表

| 大小 | 字节数 | 说明 |
|------|--------|------|
| 1GB | 1073741824 | 当前默认值 |
| 10GB | 10737418240 | 旧默认值（已废弃） |
| 100GB | 107374182400 | 可选配置 |

---

## 二、配置位置

### 2.1 数据库默认值

**位置**：`migrations/versioned/000000_init.up.sql`

```sql
storage_quota BIGINT NOT NULL DEFAULT 1073741824, -- 默认1GB配额(Bytes)
```

**说明**：
- 新建租户时，如果没有指定 `storage_quota`，将使用此默认值
- 已存在的租户不受影响，除非执行迁移更新

### 2.2 Go 代码默认值

**位置**：`internal/types/tenant.go`

```go
// Storage quota (Bytes), default is 1GB, including vector, original file, text, index, etc.
StorageQuota int64 `yaml:"storage_quota" json:"storage_quota" gorm:"default:1073741824"`
```

**说明**：
- GORM 标签中的默认值，用于新建租户时的默认值
- 必须与数据库默认值保持一致

### 2.3 数据库迁移

**位置**：`migrations/versioned/0000014_update_storage_quota_default.up.sql`

**功能**：
- 更新数据库列的默认值
- 更新现有租户的配额（从 10GB 改为 1GB）

## 三、修改方法

### 3.1 修改默认值（新建租户）

#### 方法一：修改数据库迁移文件（推荐）

1. **修改初始化迁移文件**：

```sql
-- migrations/versioned/000000_init.up.sql
storage_quota BIGINT NOT NULL DEFAULT 1073741824, -- 修改为你需要的值
```

2. **修改 Go 代码默认值**：

```go
// internal/types/tenant.go
StorageQuota int64 `yaml:"storage_quota" json:"storage_quota" gorm:"default:1073741824"`
```

3. **重新初始化数据库**（仅适用于新环境）：

```bash
# 删除并重新创建数据库
dropdb weknora
createdb weknora
./migrate up
```

#### 方法二：创建新的迁移文件（适用于已有环境）

1. **创建迁移文件**：

```bash
# 创建新的迁移文件
touch migrations/versioned/000015_update_storage_quota_to_XGB.up.sql
touch migrations/versioned/000015_update_storage_quota_to_XGB.down.sql
```

2. **编写迁移脚本**：

```sql
-- migrations/versioned/000015_update_storage_quota_to_XGB.up.sql
-- 例如：修改为 5GB (5368709120 bytes)
ALTER TABLE tenants ALTER COLUMN storage_quota SET DEFAULT 5368709120;
```

3. **执行迁移**：

```bash
./migrate up
```

### 3.2 修改现有租户的配额

#### 方法一：通过 API（推荐）

```bash
# 更新租户配额为 1GB
curl -X PUT http://localhost:8080/api/v1/tenants/{tenant_id} \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer {token}" \
  -d '{
    "storage_quota": 1073741824
  }'
```

#### 方法二：通过数据库 SQL

```sql
-- 更新特定租户的配额为 1GB
UPDATE tenants 
SET storage_quota = 1073741824 
WHERE id = {tenant_id};

-- 批量更新所有租户的配额为 1GB
UPDATE tenants 
SET storage_quota = 1073741824 
WHERE deleted_at IS NULL;
```

#### 方法三：通过前端界面

1. 登录 WeKnora 管理界面
2. 进入「租户管理」页面
3. 选择要修改的租户
4. 编辑租户信息，修改「存储配额」
5. 保存更改

## 四、从 10GB 改为 1GB 的步骤

### 4.1 已完成的修改

✅ **数据库迁移文件**：`migrations/versioned/0000014_update_storage_quota_default.up.sql`
- 更新默认值为 1GB
- 更新现有租户的配额（从 10GB 改为 1GB）

✅ **Go 代码默认值**：`internal/types/tenant.go`
- 更新 GORM 默认值为 1GB

✅ **初始化脚本**：
- `migrations/paradedb/00-init-db.sql`
- `migrations/mysql/00-init-db.sql`

### 4.2 执行步骤

#### 步骤 1：执行数据库迁移

```bash
# 执行迁移
./migrate up

# 或使用 Docker Compose
docker compose exec app ./migrate up
```

**迁移效果**：
- ✅ 数据库列的默认值更新为 1GB
- ✅ 所有使用旧默认值（10GB）的租户自动更新为 1GB
- ✅ 已自定义配额的租户不受影响

#### 步骤 2：验证修改

```sql
-- 检查默认值
SELECT column_default 
FROM information_schema.columns 
WHERE table_name = 'tenants' 
  AND column_name = 'storage_quota';

-- 应该返回：1073741824

-- 检查租户配额
SELECT id, name, storage_quota, storage_used 
FROM tenants 
WHERE deleted_at IS NULL;

-- 确认所有租户的 storage_quota 已更新为 1GB（除非之前已自定义）
```

#### 步骤 3：重启应用（可选）

```bash
# 重启应用以加载新的代码默认值
docker compose restart app
```

## 五、常见问题

### 5.1 问题：迁移后某些租户的配额仍然是 10GB

**原因**：这些租户的配额可能是手动设置的，不是使用默认值

**解决方案**：
```sql
-- 手动更新这些租户的配额
UPDATE tenants 
SET storage_quota = 1073741824 
WHERE storage_quota = 10737418240 
  AND deleted_at IS NULL;
```

### 5.2 问题：如何为特定租户设置更大的配额？

**解决方案**：通过 API 或数据库更新

```sql
-- 例如：为租户 ID 10000 设置 10GB 配额
UPDATE tenants 
SET storage_quota = 10737418240 
WHERE id = 10000;
```

### 5.3 问题：如何禁用存储配额限制？

**解决方案**：将配额设置为 0 或负数

```sql
-- 禁用配额限制（设置为 0）
UPDATE tenants 
SET storage_quota = 0 
WHERE id = {tenant_id};
```

**注意**：代码中会检查 `StorageQuota > 0`，所以设置为 0 表示无限制。

### 5.4 问题：如何查看租户的存储使用情况？

**查询 SQL**：
```sql
SELECT 
    id,
    name,
    storage_quota,
    storage_used,
    ROUND(storage_used::numeric / storage_quota::numeric * 100, 2) as usage_percentage
FROM tenants 
WHERE deleted_at IS NULL
ORDER BY storage_used DESC;
```

## 七、最佳实践

### 7.1 配额设置建议

| 环境类型 | 推荐配额 | 说明 |
|---------|---------|------|
| **开发环境** | 1GB | 足够用于开发和测试 |
| **测试环境** | 5GB | 用于功能测试和集成测试 |
| **生产环境** | 10GB+ | 根据实际需求设置 |

### 7.2 监控建议

1. **定期检查存储使用率**：
   ```sql
   SELECT 
       id, name,
       storage_used, storage_quota,
       ROUND(storage_used::numeric / storage_quota::numeric * 100, 2) as usage_pct
   FROM tenants 
   WHERE deleted_at IS NULL 
     AND storage_quota > 0
   ORDER BY usage_pct DESC;
   ```

2. **设置告警阈值**：
   - 使用率 > 80%：发送警告
   - 使用率 > 95%：发送紧急告警

3. **定期清理**：
   - 删除不需要的知识库和文档
   - 清理过期的向量索引
