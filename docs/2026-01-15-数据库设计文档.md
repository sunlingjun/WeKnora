## 概述

WeKnora 使用 **PostgreSQL** 作为主数据库，支持多租户架构，采用软删除（Soft Delete）机制。数据库设计遵循以下原则：

- ✅ **多租户隔离**：通过 `tenant_id` 实现数据隔离
- ✅ **软删除**：所有表都包含 `deleted_at` 字段
- ✅ **时间戳追踪**：`created_at` 和 `updated_at` 自动维护
- ✅ **JSONB 灵活配置**：大量使用 JSONB 存储配置信息
- ✅ **向量搜索支持**：集成 pgvector 扩展用于向量相似度搜索

```
├─────────────────┤
│   MCP 服务       │  mcp_services
└─────────────────┘
```

## 表结构详解

### 1. 租户表 (tenants)

**用途**：存储租户（客户）信息，支持多租户 SaaS 架构

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | SERIAL | PRIMARY KEY | 租户 ID，从 10000 开始 |
| `name` | VARCHAR(255) | NOT NULL | 租户名称 |
| `description` | TEXT | | 租户描述 |
| `api_key` | VARCHAR(64) | NOT NULL | API 密钥（用于 API 认证） |
| `retriever_engines` | JSONB | NOT NULL DEFAULT '[]' | 检索引擎配置列表 |
| `status` | VARCHAR(50) | DEFAULT 'active' | 租户状态（active/inactive） |
| `business` | VARCHAR(255) | NOT NULL | 业务标识 |
| `storage_quota` | BIGINT | NOT NULL DEFAULT 10737418240 | 存储配额（字节），默认 10GB |
| `storage_used` | BIGINT | NOT NULL DEFAULT 0 | 已使用存储（字节） |
| `agent_config` | JSONB | DEFAULT NULL | 租户级智能体配置 |
| `context_config` | JSONB | | 全局上下文配置（所有会话的默认值） |
| `conversation_config` | JSONB | | 全局对话配置（普通模式会话的默认值） |
| `web_search_config` | JSONB | DEFAULT NULL | 联网搜索配置 |
| `created_at` | TIMESTAMPTZ | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMPTZ | DEFAULT CURRENT_TIMESTAMP | 更新时间 |
| `deleted_at` | TIMESTAMPTZ | | 软删除时间 |

**索引**：
- `idx_tenants_api_key` ON `api_key`
- `idx_tenants_status` ON `status`
- `idx_tenants_agent_config` USING GIN (`agent_config`)

**特殊说明**：
- 租户 ID 序列从 10000 开始，避免与系统保留 ID 冲突
- `agent_config` 存储租户级别的智能体配置（已迁移到 `custom_agents` 表）

### 2. 用户表 (users)

**用途**：存储用户账户信息，支持用户认证

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | VARCHAR(36) | PRIMARY KEY | 用户 ID（UUID） |
| `username` | VARCHAR(100) | NOT NULL UNIQUE | 用户名 |
| `email` | VARCHAR(255) | NOT NULL UNIQUE | 邮箱地址 |
| `password_hash` | VARCHAR(255) | NOT NULL | 密码哈希值 |
| `avatar` | VARCHAR(500) | | 头像 URL |
| `tenant_id` | INTEGER | | 所属租户 ID |
| `can_access_all_tenants` | BOOLEAN | NOT NULL DEFAULT FALSE | 是否可访问所有租户 |
| `is_active` | BOOLEAN | NOT NULL DEFAULT TRUE | 是否激活 |
| `created_at` | TIMESTAMPTZ | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMPTZ | DEFAULT CURRENT_TIMESTAMP | 更新时间 |
| `deleted_at` | TIMESTAMPTZ | | 软删除时间 |

**索引**：
- `idx_users_username` ON `username`
- `idx_users_email` ON `email`
- `idx_users_tenant_id` ON `tenant_id`
- `idx_users_deleted_at` ON `deleted_at`

### 8. 向量嵌入表 (embeddings)

**用途**：存储向量嵌入数据，支持向量相似度搜索（使用 pgvector）

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | SERIAL | PRIMARY KEY | 嵌入 ID（自增） |
| `source_id` | VARCHAR(64) | NOT NULL | 源 ID（通常是 chunk_id） |
| `source_type` | INTEGER | NOT NULL | 源类型（1=chunk） |
| `chunk_id` | VARCHAR(64) | | 分块 ID |
| `knowledge_id` | VARCHAR(64) | | 知识 ID |
| `knowledge_base_id` | VARCHAR(64) | | 知识库 ID |
| `content` | TEXT | | 原始内容 |
| `dimension` | INTEGER | NOT NULL | 向量维度 |
| `embedding` | halfvec | | 向量嵌入（半精度向量） |
| `is_enabled` | BOOLEAN | DEFAULT TRUE | 是否启用 |
| `created_at` | TIMESTAMPTZ | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | TIMESTAMPTZ | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

**索引**：
- `embeddings_unique_source` UNIQUE ON `source_id, source_type`
- `embeddings_search_idx` USING bm25（全文搜索索引，支持中文分词）
- `embeddings_embedding_idx_3584` USING hnsw（HNSW 向量索引，维度 3584）
- `embeddings_embedding_idx_798` USING hnsw（HNSW 向量索引，维度 798）
- `idx_embeddings_is_enabled` ON `is_enabled`
- `idx_embeddings_knowledge_base_id` ON `knowledge_base_id`

**特殊说明**：
- 此表仅在 PostgreSQL + pgvector 环境下创建（条件迁移）
- 使用 `halfvec` 类型存储半精度向量，节省存储空间
- 支持 BM25 全文搜索和 HNSW 向量搜索
- 中文分词使用 `chinese_lindera` tokenizer

## 索引设计

### B-Tree 索引

用于等值查询和范围查询：

- **租户相关**：`tenant_id` 字段上的索引（几乎所有表）
- **外键字段**：`knowledge_base_id`、`knowledge_id`、`session_id` 等
- **状态字段**：`status`、`parse_status`、`enable_status` 等
- **时间字段**：`deleted_at`、`expires_at` 等

### GIN 索引

用于 JSONB 字段的查询：

- `idx_tenants_agent_config` ON `tenants(agent_config)`
- `idx_sessions_agent_config` ON `sessions(agent_config)`
- `idx_sessions_context_config` ON `sessions(context_config)`
- `idx_messages_agent_steps` ON `messages(agent_steps)`

### HNSW 向量索引

用于向量相似度搜索（pgvector）：

- `embeddings_embedding_idx_3584`：维度 3584 的向量索引
- `embeddings_embedding_idx_798`：维度 798 的向量索引

### BM25 全文搜索索引

用于中文全文搜索（ParadeDB）：

- `embeddings_search_idx`：支持中文分词的全文搜索索引

## 设计模式

### 1. 软删除（Soft Delete）

所有表都包含 `deleted_at` 字段：
- `NULL`：记录未删除
- 非 `NULL`：记录已删除（时间戳）

**优势**：
- 数据可恢复
- 审计追踪
- 关联数据完整性

### 2. 多租户隔离

通过 `tenant_id` 实现数据隔离：
- 所有业务表都包含 `tenant_id`
- 查询时自动过滤租户数据
- 租户 ID 从 10000 开始，避免与系统 ID 冲突

### 3. JSONB 灵活配置

大量使用 JSONB 存储配置：
- `agent_config`：智能体配置
- `context_config`：上下文配置
- `parameters`：模型参数
- `metadata`：元数据

**优势**：
- 灵活扩展，无需频繁修改表结构
- 支持部分更新
- 支持索引（GIN）

### 4. 时间戳自动维护

- `created_at`：创建时间（默认值）
- `updated_at`：更新时间（部分表使用触发器自动更新）

### 5. 条件迁移

`embeddings` 表使用条件迁移：
- 仅在 PostgreSQL + pgvector 环境下创建
- 通过 `app.skip_embedding` 配置控制

## 性能优化建议

### 1. 索引优化

- ✅ 已为常用查询字段创建索引
- ✅ JSONB 字段使用 GIN 索引
- ✅ 向量字段使用 HNSW 索引

### 2. 查询优化

- 使用 `tenant_id` 过滤，利用索引
- 软删除查询时添加 `deleted_at IS NULL` 条件
- JSONB 查询使用 GIN 索引支持的运算符

### 3. 存储优化

- 使用 `halfvec` 存储向量，节省空间
- 大文本字段使用 `TEXT` 类型
- JSONB 字段压缩存储
