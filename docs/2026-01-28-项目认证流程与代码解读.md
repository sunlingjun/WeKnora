## 二、认证流程概述

WeKnora 支持三种认证方式：

1. **JWT Token 认证**：用户登录后获取的 Bearer Token
2. **X-API-Key 认证**：租户级别的 API Key（`sk-` 开头）
3. **CAS 单点登录**：企业级 SSO 认证

### 2.1 认证中间件位置

认证中间件在路由注册时被添加到全局中间件链中：

```go
// internal/router/router.go
r.Use(middleware.Auth(params.TenantService, params.UserService, params.Config))
```

### 2.2 无需认证的 API

以下 API 不需要认证：

```go
var noAuthAPI = map[string][]string{
    "/health":               {"GET"},
    "/api/v1/auth/register": {"POST"},
    "/api/v1/auth/login":    {"POST"},
    "/api/v1/auth/refresh":  {"POST"},
}
```

## 三、JWT Token 认证流程

### 3.1 用户注册流程

**API**: `POST /api/v1/auth/register`

**代码路径**: `internal/handler/auth.go::Register`

**流程**：

1. **检查注册是否启用**
   ```go
   if os.Getenv("DISABLE_REGISTRATION") == "true" {
       return errors.NewForbiddenError("Registration is disabled")
   }
   ```

2. **解析请求参数**
   ```go
   var req types.RegisterRequest
   c.ShouldBindJSON(&req)
   ```

3. **调用服务层注册用户**
   ```go
   user, err := h.userService.Register(ctx, &req)
   ```

4. **服务层处理**（`internal/application/service/user.go::Register`）
   - 验证输入（用户名、邮箱、密码）
   - 检查用户是否已存在（按邮箱和用户名）
   - 使用 bcrypt 加密密码
   - **自动创建默认租户**（重要！）
   - 创建用户记录
   - 返回用户信息

### 3.2 用户登录流程

**API**: `POST /api/v1/auth/login`

**代码路径**: `internal/handler/auth.go::Login`

**流程**：

1. **解析登录请求**
   ```go
   var req types.LoginRequest
   c.ShouldBindJSON(&req)
   ```

2. **调用服务层登录**
   ```go
   response, err := h.userService.Login(ctx, &req)
   ```

3. **服务层处理**（`internal/application/service/user.go::Login`）
   ```go
   // 1. 根据邮箱查找用户
   user, err := s.userRepo.GetUserByEmail(ctx, req.Email)
   
   // 2. 检查用户是否激活
   if !user.IsActive {
       return "Account is disabled"
   }
   
   // 3. 验证密码（bcrypt 比较）
   err = bcrypt.CompareHashAndPassword([]byte(user.PasswordHash), []byte(req.Password))
   
   // 4. 生成 JWT Token
   accessToken, refreshToken, err := s.GenerateTokens(ctx, user)
   
   // 5. 获取租户信息
   tenant, err := s.tenantService.GetTenantByID(ctx, user.TenantID)
   ```

4. **Token 生成**（`internal/application/service/user.go::GenerateTokens`）
   ```go
   // Access Token（24小时有效期）
   accessClaims := jwt.MapClaims{
       "user_id":   user.ID,
       "email":     user.Email,
       "tenant_id": user.TenantID,
       "exp":       time.Now().Add(24 * time.Hour).Unix(),
       "iat":       time.Now().Unix(),
       "type":      "access",
   }
   
   // Refresh Token（7天有效期）
   refreshClaims := jwt.MapClaims{
       "user_id": user.ID,
       "exp":     time.Now().Add(7 * 24 * time.Hour).Unix(),
       "iat":     time.Now().Unix(),
       "type":    "refresh",
   }
   
   // 签名并存储到数据库
   ```

5. **返回响应**
   ```json
   {
       "success": true,
       "message": "Login successful",
       "user": {...},
       "tenant": {...},
       "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
       "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
   }
   ```

### 3.3 Token 验证流程（认证中间件）

**代码路径**: `internal/middleware/auth.go::Auth`

**流程**：

```go
func Auth(...) gin.HandlerFunc {
    return func(c *gin.Context) {
        // 1. 检查是否无需认证
        if isNoAuthAPI(c.Request.URL.Path, c.Request.Method) {
            c.Next()
            return
        }

        // 2. 尝试 JWT Token 认证
        authHeader := c.GetHeader("Authorization")
        if authHeader != "" && strings.HasPrefix(authHeader, "Bearer ") {
            token := strings.TrimPrefix(authHeader, "Bearer ")
            
            // 3. 验证 Token
            user, err := userService.ValidateToken(c.Request.Context(), token)
            if err == nil && user != nil {
                // 4. 检查跨租户访问（X-Tenant-ID 头）
                targetTenantID := user.TenantID
                tenantHeader := c.GetHeader("X-Tenant-ID")
                if tenantHeader != "" {
                    // 验证用户是否有跨租户权限
                    if canAccessTenant(user, parsedTenantID, cfg) {
                        targetTenantID = parsedTenantID
                    }
                }

                // 5. 获取租户信息
                tenant, err := tenantService.GetTenantByID(ctx, targetTenantID)

                // 6. 存储到 Context
                c.Set(types.TenantIDContextKey.String(), targetTenantID)
                c.Set(types.TenantInfoContextKey.String(), tenant)
                c.Set(types.UserIDContextKey.String(), user.ID)
                c.Set(types.UserInfoContextKey.String(), user)
                c.Request = c.Request.WithContext(...)

                c.Next()
                return
            }
        }

        // 7. 尝试 X-API-Key 认证（见下一节）
        // ...
    }
}
```

**Token 验证逻辑**（`internal/application/service/user.go::ValidateToken`）：

```go
func (s *userService) ValidateToken(ctx context.Context, tokenString string) (*types.User, error) {
    // 1. 解析 JWT Token
    token, err := jwt.Parse(tokenString, func(token *jwt.Token) (interface{}, error) {
        // 验证签名算法
        if _, ok := token.Method.(*jwt.SigningMethodHMAC); !ok {
            return nil, fmt.Errorf("unexpected signing method")
        }
        return []byte(getJwtSecret()), nil
    })

    // 2. 提取 Claims
    claims, ok := token.Claims.(jwt.MapClaims)
    userID := claims["user_id"].(string)

    // 3. 检查 Token 是否被撤销（查询数据库）
    tokenRecord, err := s.tokenRepo.GetTokenByValue(ctx, tokenString)
    if tokenRecord == nil || tokenRecord.IsRevoked {
        return nil, errors.New("token is revoked")
    }

    // 4. 获取用户信息
    return s.userRepo.GetUserByID(ctx, userID)
}
```

## 四、X-API-Key 认证流程

### 4.1 API Key 格式

API Key 格式：`sk-{tenant_id}-{random_string}`

例如：`sk-10001-abc123def456`

### 4.2 认证流程（认证中间件）

**代码路径**: `internal/middleware/auth.go::Auth`

**流程**：

```go
// 1. 提取 X-API-Key 头
apiKey := c.GetHeader("X-API-Key")
if apiKey != "" {
    // 2. 从 API Key 提取租户 ID
    tenantID, err := tenantService.ExtractTenantIDFromAPIKey(apiKey)
    // 提取逻辑：解析 "sk-{tenant_id}-..." 格式

    // 3. 验证 API Key 有效性（查询数据库）
    tenant, err := tenantService.GetTenantByID(ctx, tenantID)
    if tenant == nil || tenant.APIKey != apiKey {
        return "Unauthorized: invalid API key"
    }

    // 4. 存储租户信息到 Context（注意：没有用户信息）
    c.Set(types.TenantIDContextKey.String(), tenantID)
    c.Set(types.TenantInfoContextKey.String(), tenant)
    c.Request = c.Request.WithContext(...)

    c.Next()
    return
}
```

### 4.3 API Key 的限制

- **仅设置租户信息**：X-API-Key 认证只设置 `TenantID` 和 `TenantInfo`，不设置用户信息
- **不支持用户相关 API**：某些需要用户信息的 API（如 `/api/v1/auth/me`）不支持 X-API-Key 认证
- **租户级别认证**：适用于服务间调用、API 集成等场景

## 五、CAS 单点登录认证流程

### 5.1 CAS 认证概述

CAS（Central Authentication Service）是一种企业级单点登录（SSO）协议。WeKnora 集成了 CAS 认证，支持通过 Cookie 中的 CAS 会话信息进行认证。

### 5.2 CAS 认证流程

**API**: `GET /api/v1/cas/validate`

**代码路径**: `internal/handler/cas_auth.go::ValidateCASSession`

**流程**：

```go
func (h *CASAuthHandler) ValidateCASSession(c *gin.Context) {
    // 1. 获取 CAS 配置（根据环境）
    casConfig := h.config.CAS.GetCurrentConfig()
    cookieSID := casConfig.CookieSID  // 例如：_cas_sid_test
    cookieUID := casConfig.CookieUID  // 例如：_cas_uid_test

    // 2. 从 Cookie 读取 CAS 会话信息
    casSid, _ := c.Cookie(cookieSID)
    casUid, _ := c.Cookie(cookieUID)
    
    if casSid == "" || casUid == "" {
        return "未获取到登录信息"
    }

    // 3. 获取 Referer 头（CAS API 需要）
    referer := c.GetHeader("Referer")
    if referer == "" {
        referer = fmt.Sprintf("https://%s/", casConfig.LoginHost)
    }

    // 4. 验证 CAS 会话（调用 CAS 服务器 API）
    casUserInfo, err := h.casAuthService.ValidateCASSession(ctx, casSid, casUid, referer)
    // 内部逻辑：
    // - 构建 CAS API URL
    // - 发送 HTTP 请求（携带 Cookie 和 Referer）
    // - 解析响应，提取用户信息（id, loginName, realName, mobilePhone）

    // 5. 自动绑定或创建用户
    user, err := h.casAuthService.AutoBindUser(ctx, casUserInfo)
    // 内部逻辑：
    // - 按优先级查找用户：cas_user_id → email
    // - 如果用户不存在，创建新用户（处理用户名/邮箱冲突）
    // - 如果用户存在，更新 CAS 相关字段

    // 6. 自动绑定或创建租户
    tenant, err := h.casAuthService.AutoBindTenant(ctx, casUserInfo, user)
    // 内部逻辑：
    // - 检查用户是否已有租户
    // - 如果用户没有租户，创建新租户并绑定

    // 7. 生成 JWT Token（用于后续 API 调用）
    token, refreshToken, err := h.userService.GenerateTokens(ctx, user)

    // 8. 返回响应
    {
        "code": 0,
        "data": {
            "user": {...},
            "tenant": {...},
            "token": "...",
            "refresh_token": "..."
        }
    }
}
```

### 5.3 CAS 用户自动绑定逻辑

**代码路径**: `internal/application/service/cas_auth.go::AutoBindUser`

**流程**：

```go
func (s *casAuthService) AutoBindUser(ctx context.Context, casUserInfo *types.CASUserInfo) (*types.User, error) {
    // 1. 优先按 cas_user_id 查找用户
    if casUserInfo.ID != "" {
        user, _ := s.userRepo.GetUserByCASUserID(ctx, casUserInfo.ID)
        if user != nil {
            // 更新 CAS 相关字段
            user.CASLoginName = casUserInfo.LoginName
            user.CASRealName = casUserInfo.RealName
            user.CASMobilePhone = casUserInfo.MobilePhone
            s.userRepo.UpdateUser(ctx, user)
            return user, nil
        }
    }

    // 2. 按邮箱查找用户
    if casUserInfo.Email != "" {
        user, _ := s.userRepo.GetUserByEmail(ctx, casUserInfo.Email)
        if user != nil {
            // 绑定 CAS 信息
            user.CASUserID = casUserInfo.ID
            // ... 更新其他字段
            s.userRepo.UpdateUser(ctx, user)
            return user, nil
        }
    }

    // 3. 创建新用户
    // - 处理用户名冲突（添加后缀）
    // - 处理邮箱冲突（使用 CAS 邮箱或生成）
    // - 生成随机密码（用户无法直接登录，只能通过 CAS）
    user := &types.User{
        ID:           uuid.New().String(),
        Username:     username,
        Email:        email,
        PasswordHash: hashedPassword, // 随机密码
        CASUserID:    casUserInfo.ID,
        CASLoginName: casUserInfo.LoginName,
        CASRealName:  casUserInfo.RealName,
        CASMobilePhone: casUserInfo.MobilePhone,
        IsActive:     true,
    }
    s.userRepo.CreateUser(ctx, user)
    return user, nil
}
```

### 5.4 CAS 租户自动绑定逻辑

**代码路径**: `internal/application/service/cas_auth.go::AutoBindTenant`

**流程**：

```go
func (s *casAuthService) AutoBindTenant(ctx context.Context, casUserInfo *types.CASUserInfo, user *types.User) (*types.Tenant, error) {
    // 1. 检查用户是否已有租户
    if user.TenantID > 0 {
        tenant, err := s.tenantService.GetTenantByID(ctx, user.TenantID)
        if err == nil && tenant != nil {
            return tenant, nil
        }
    }

    // 2. 创建新租户
    tenant := &types.Tenant{
        Name:        fmt.Sprintf("%s's Workspace", casUserInfo.RealName),
        Description: "CAS 用户默认工作空间",
        Status:      "active",
    }
    createdTenant, err := s.tenantService.CreateTenant(ctx, tenant)

    // 3. 绑定用户到租户
    user.TenantID = createdTenant.ID
    s.userService.UpdateUser(ctx, user)

    return createdTenant, nil
}
```

## 七、代码详细解读

### 7.1 认证中间件详解

**文件**: `internal/middleware/auth.go`

**关键函数**：

#### `isNoAuthAPI(path, method string) bool`
- **作用**：检查 API 是否在无需认证列表中
- **实现**：支持前缀匹配（以 `*` 结尾）和精确匹配

#### `canAccessTenant(user, targetTenantID, cfg) bool`
- **作用**：检查用户是否有跨租户访问权限
- **条件**：
  1. 功能已启用（`cfg.Tenant.EnableCrossTenantAccess == true`）
  2. 用户有跨租户权限（`user.CanAccessAllTenants == true`）
  3. 或目标租户是用户自己的租户

#### `Auth(tenantService, userService, cfg) gin.HandlerFunc`
- **作用**：认证中间件主函数
- **认证优先级**：
  1. JWT Token（`Authorization: Bearer {token}`）
  2. X-API-Key（`X-API-Key: sk-...`）
  3. 如果都没有，返回 401

### 7.2 用户服务详解

**文件**: `internal/application/service/user.go`

**关键函数**：

#### `Register(ctx, req) (*User, error)`
- **流程**：
  1. 验证输入（用户名、邮箱、密码）
  2. 检查用户是否已存在（按邮箱和用户名）
  3. 使用 bcrypt 加密密码（`bcrypt.GenerateFromPassword`）
  4. **创建默认租户**（重要！每个用户都有独立的租户）
  5. 创建用户记录
  6. 返回用户信息

#### `Login(ctx, req) (*LoginResponse, error)`
- **流程**：
  1. 根据邮箱查找用户
  2. 检查用户是否激活
  3. 验证密码（`bcrypt.CompareHashAndPassword`）
  4. 生成 JWT Token（Access Token + Refresh Token）
  5. 获取租户信息
  6. 返回登录响应

#### `GenerateTokens(ctx, user) (accessToken, refreshToken, error)`
- **流程**：
  1. 创建 Access Token Claims（24小时有效期）
  2. 创建 Refresh Token Claims（7天有效期）
  3. 使用 HS256 算法签名（密钥从环境变量获取）
  4. **存储 Token 到数据库**（`auth_tokens` 表）
  5. 返回 Token 字符串

#### `ValidateToken(ctx, tokenString) (*User, error)`
- **流程**：
  1. 解析 JWT Token（验证签名算法和签名）
  2. 提取 Claims（`user_id`, `email`, `tenant_id`）
  3. **检查 Token 是否被撤销**（查询数据库）
  4. 根据 `user_id` 获取用户信息
  5. 返回用户对象

#### `RefreshToken(ctx, refreshTokenString) (accessToken, newRefreshToken, error)`
- **流程**：
  1. 解析 Refresh Token
  2. 验证 Token 类型为 `"refresh"`
  3. 检查 Token 是否被撤销
  4. **撤销旧的 Refresh Token**（设置 `IsRevoked = true`）
  5. 生成新的 Access Token 和 Refresh Token
  6. 返回新 Token

### 7.3 CAS 认证服务详解

**文件**: `internal/application/service/cas_auth.go`

**关键函数**：

#### `ValidateCASSession(ctx, casSid, casUid, referer) (*CASUserInfo, error)`
- **流程**：
  1. 构建 CAS API URL（根据环境配置）
  2. 创建 HTTP 请求（携带 Cookie 和 Referer）
  3. 发送请求到 CAS 服务器
  4. 解析响应 JSON，提取用户信息
  5. 返回 `CASUserInfo` 对象

#### `AutoBindUser(ctx, casUserInfo) (*User, error)`
- **查找优先级**：
  1. 按 `cas_user_id` 查找
  2. 按 `email` 查找
  3. 如果都不存在，创建新用户
- **冲突处理**：
  - 用户名冲突：添加数字后缀（`username_1`, `username_2`）
  - 邮箱冲突：使用 CAS 邮箱或生成临时邮箱

#### `AutoBindTenant(ctx, casUserInfo, user) (*Tenant, error)`
- **流程**：
  1. 检查用户是否已有租户
  2. 如果已有租户且存在，返回该租户
  3. 如果用户没有租户，创建新租户
  4. 绑定用户到租户（更新 `user.TenantID`）

## 八、安全考虑

### 8.1 JWT Token 安全

1. **密钥管理**：JWT Secret 从环境变量获取（`JWT_SECRET`）
2. **Token 撤销**：Token 存储在数据库中，支持撤销机制
3. **过期时间**：Access Token 24小时，Refresh Token 7天
4. **签名算法**：使用 HS256（HMAC-SHA256）

### 8.2 API Key 安全

1. **格式验证**：API Key 必须符合 `sk-{tenant_id}-{random}` 格式
2. **数据库验证**：API Key 必须与数据库中的租户 API Key 匹配
3. **租户隔离**：API Key 认证后只能访问对应租户的数据

### 8.3 CAS 认证安全

1. **Cookie 验证**：必须同时提供 `_cas_sid` 和 `_cas_uid` Cookie
2. **Referer 验证**：CAS 服务器会验证 Referer 头
3. **自动绑定**：用户和租户自动绑定，避免手动配置错误

### 8.4 密码安全

1. **加密算法**：使用 bcrypt（成本因子 10）
2. **密码验证**：登录时使用 `bcrypt.CompareHashAndPassword` 比较
3. **随机密码**：CAS 用户创建时生成随机密码（用户无法直接登录）

## 九、常见问题

### 9.1 为什么注册时自动创建租户？

**原因**：WeKnora 采用多租户架构，每个用户都需要属于一个租户。注册时自动创建租户可以：
- 简化用户注册流程
- 确保每个用户都有独立的工作空间
- 支持数据隔离

### 9.2 JWT Token 和 Refresh Token 的区别？

- **Access Token**：用于 API 调用，有效期短（24小时），包含用户和租户信息
- **Refresh Token**：用于刷新 Access Token，有效期长（7天），只包含用户 ID

### 9.3 X-API-Key 认证为什么没有用户信息？

**原因**：X-API-Key 是租户级别的认证，用于服务间调用和 API 集成。这些场景不需要用户信息，只需要租户信息即可。

### 9.4 CAS 认证后为什么还要生成 JWT Token？

**原因**：CAS 认证只验证了用户的登录状态，但后续的 API 调用需要使用 JWT Token。这样可以：
- 统一认证机制（所有 API 都使用 JWT Token）
- 支持 Token 撤销和过期管理
- 减少对 CAS 服务器的依赖

## 十、总结

### 10.1 认证方式对比

| 认证方式 | 适用场景 | 设置的信息 | 支持的功能 |
|---------|---------|-----------|-----------|
| JWT Token | 用户登录后使用 | 用户信息 + 租户信息 | 所有功能 |
| X-API-Key | 服务间调用、API 集成 | 仅租户信息 | 租户级别功能 |
| CAS SSO | 企业单点登录 | 用户信息 + 租户信息（自动绑定） | 所有功能 |

### 10.2 关键流程总结

1. **项目启动**：依赖注入 → 初始化服务 → 注册路由 → 启动 HTTP 服务器
2. **用户注册**：验证输入 → 创建租户 → 创建用户 → 返回用户信息
3. **用户登录**：验证密码 → 生成 Token → 返回 Token 和用户信息
4. **请求认证**：检查 Token/API Key → 验证有效性 → 设置 Context → 继续处理
5. **CAS 认证**：读取 Cookie → 验证 CAS 会话 → 自动绑定用户/租户 → 生成 JWT Token
