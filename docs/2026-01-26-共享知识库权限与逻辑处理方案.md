# 共享知识库权限与逻辑处理方案

> **文档创建**：2026-01-28

## 📋 问题分析

### 1. 文件数量统计问题
**问题**：我加入的共享知识库，在知识库列表内文件数量不对

**原因**：
- `ListUserKnowledgeBases` handler 中使用当前用户的 `tenantID` 统计文档数量
- 共享知识库可能属于不同租户，导致统计不准确

**解决方案**：
- 使用知识库自身的 `kb.TenantID` 进行统计，而不是当前用户的 `tenantID`

### 2. 知识库详情页权限问题
**问题**：我加入的共享知识库点击进入详情，使用的是现有的知识库详情，没有权限区分

**需要实现的权限逻辑**：
1. **创建者（Owner）**：拥有全部权限
   - 查看、编辑、删除文档
   - 修改知识库设置
   - 管理成员

2. **编辑者（Editor）**：拥有编辑权限
   - 查看、编辑文档
   - 上传新文档
   - 删除文档（可选）
   - **不能**修改知识库设置
   - **不能**管理成员

3. **查看者（Viewer）**：只有查看权限
   - 查看文档
   - **不能**编辑、删除文档
   - **不能**上传文档
   - **不能**修改知识库设置
   - **不能**管理成员

### 3. 知识库设置页面权限问题
**问题**：共享知识库设置只有创建者有设置权限

**需要实现**：
- 设置页面入口权限检查（只有创建者可见）
- 设置页面内所有操作权限检查
- 左侧菜单增加"成员管理"（只有创建者可见）

## 🔧 实现方案

### 阶段一：修复文件数量统计问题

#### 1.1 后端修改
**文件**：`internal/handler/knowledgebase.go`

**修改点**：`ListUserKnowledgeBases` handler

```go
// 修改前：使用当前用户的 tenantID
count, err := knowledgeRepo.CountKnowledgeByKnowledgeBaseID(ctx, tenantID, kb.ID)

// 修改后：使用知识库的 tenantID
count, err := knowledgeRepo.CountKnowledgeByKnowledgeBaseID(ctx, kb.TenantID, kb.ID)
```

### 阶段二：知识库详情页权限控制

#### 2.1 后端权限检查中间件/方法

**新增文件**：`internal/middleware/kb_permission.go`

**功能**：
- 检查用户对知识库的访问权限
- 返回用户角色（owner/editor/viewer）
- 根据操作类型（read/write/delete/settings）检查权限

#### 2.2 前端权限状态管理

**修改文件**：`frontend/src/views/knowledge/KnowledgeBase.vue`

**需要添加**：
- 获取知识库权限信息（`is_owner`, `member_role`）
- 根据权限控制UI显示：
  - 上传按钮（editor/viewer 不可见）
  - 编辑按钮（viewer 不可见）
  - 删除按钮（viewer 不可见）
  - 设置按钮（只有 owner 可见）

#### 2.3 后端API权限检查

**修改文件**：
- `internal/handler/knowledge.go` - 文档操作权限检查
- `internal/handler/knowledgebase.go` - 知识库操作权限检查

### 阶段三：知识库设置页面权限控制

#### 3.1 设置页面入口权限

**修改文件**：`frontend/src/views/knowledge/KnowledgeBase.vue`

**修改点**：`handleOpenKBSettings` 方法
- 检查是否是创建者
- 非创建者提示无权限

#### 3.2 设置页面内容权限

**修改文件**：`frontend/src/views/knowledge/KnowledgeBaseEditorModal.vue`

**需要添加**：
- 权限检查逻辑
- 根据权限禁用/隐藏相关设置项
- 共享知识库的创建者才能修改设置

#### 3.3 设置页面成员管理菜单

**修改文件**：`frontend/src/views/knowledge/KnowledgeBaseEditorModal.vue`

**需要添加**：
- 左侧菜单项："成员管理"（只有创建者可见）
- 成员管理页面内容（复用 `KnowledgeBaseMembers.vue` 组件）

### 阶段四：后端API权限验证

#### 4.1 文档操作权限检查

**修改文件**：
- `internal/handler/knowledge.go`
  - `CreateKnowledge` - 检查 write 权限
  - `UpdateKnowledge` - 检查 write 权限
  - `DeleteKnowledge` - 检查 write/delete 权限
  - `ListKnowledge` - 检查 read 权限

#### 4.2 知识库操作权限检查

**修改文件**：
- `internal/handler/knowledgebase.go`
  - `UpdateKnowledgeBase` - 检查 settings 权限（只有 owner）
  - `GetKnowledgeBase` - 检查 read 权限

## 📝 权限模型

### 权限矩阵

| 操作 | Owner | Editor | Viewer |
|------|-------|--------|--------|
| 查看文档 | ✅ | ✅ | ✅ |
| 上传文档 | ✅ | ✅ | ❌ |
| 编辑文档 | ✅ | ✅ | ❌ |
| 删除文档 | ✅ | ✅ | ❌ |
| 修改设置 | ✅ | ❌ | ❌ |
| 管理成员 | ✅ | ❌ | ❌ |

### 权限检查方法

**后端**：`internal/application/service/shared_kb.go`

```go
// CheckMemberPermission 检查成员权限
func (s *sharedKnowledgeBaseService) CheckMemberPermission(
    ctx context.Context,
    kbID string,
    operation string, // "read", "write", "delete", "settings"
) (bool, error)
```

## 🎯 实施优先级

1. **高优先级**：
   - ✅ 修复文件数量统计问题
   - ✅ 知识库详情页权限控制（上传/编辑/删除按钮）
   - ✅ 设置页面入口权限检查

2. **中优先级**：
   - 设置页面内容权限控制
   - 设置页面成员管理菜单

3. **低优先级**：
   - 后端API权限验证（已有基础检查，需要完善）
   - 权限错误提示优化

## 📌 注意事项

1. **向后兼容**：个人知识库（`visibility='private'`）保持原有逻辑，不受影响
2. **权限缓存**：考虑缓存用户权限信息，避免频繁查询数据库
3. **错误提示**：权限不足时，提供清晰的错误提示
4. **UI反馈**：权限不足的操作应该禁用或隐藏，而不是点击后报错
