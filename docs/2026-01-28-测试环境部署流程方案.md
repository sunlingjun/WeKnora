## 一、环境准备

### 1.1 服务器配置

| 资源项 | 配置 | 说明 |
|--------|------|------|
| **CPU** | 8 核 | 满足测试环境需求 |
| **内存** | 16GB | 充足的内存资源 |
| **存储** | 100GB SSD | 数据库+MinIO+日志存储 |
| **操作系统** | Linux (Ubuntu 20.04+ / CentOS 7+) | 推荐 Ubuntu 22.04 LTS |
| **网络** | 内网/公网 | 根据实际需求 |

### 1.2 系统依赖安装

#### 1.2.1 安装 Docker 和 Docker Compose

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER

# 安装 Docker Compose V2
sudo apt-get update
sudo apt-get install docker-compose-plugin

# 验证安装
docker --version
docker compose version
```

#### 1.2.2 安装 Git

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install git -y

# CentOS/RHEL
sudo yum install git -y
```

#### 1.2.3 安装构建工具（用于构建 APP 和 FRONTEND 镜像）

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install -y \
    build-essential \
    curl \
    wget \
    git \
    make

# 安装 Go（用于构建 APP 镜像）
wget https://go.dev/dl/go1.21.linux-amd64.tar.gz
sudo tar -C /usr/local -xzf go1.21.linux-amd64.tar.gz
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.bashrc
source ~/.bashrc

# 安装 Node.js（用于构建 FRONTEND 镜像）
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证安装
go version
node --version
npm --version
```

### 1.3 创建部署目录

```bash
# 创建项目目录
sudo mkdir -p /opt/weknora
sudo chown $USER:$USER /opt/weknora
cd /opt/weknora

# 创建数据目录
sudo mkdir -p /data/weknora/{postgres,redis,minio,neo4j,logs}
sudo chown -R $USER:$USER /data/weknora
```

## 二、镜像构建策略

### 2.1 镜像构建方案

| 服务 | 镜像来源 | 说明 |
|------|---------|------|
| **APP** | 本地构建 | 使用 `docker/Dockerfile.app` |
| **FRONTEND** | 本地构建 | 使用 `frontend/Dockerfile` |
| **PostgreSQL** | 开源镜像 | `paradedb/paradedb:v0.21.4-pg17` |
| **Redis** | 开源镜像 | `redis:7.0-alpine` |
| **MinIO** | 开源镜像 | `minio/minio:RELEASE.2025-09-07T16-13-09Z` |
| **Neo4j** | 开源镜像 | `neo4j:2025.10.1` |
| **Docreader** | 开源镜像 | `wechatopenai/weknora-docreader:latest`（从 Docker Hub 拉取） |

### 2.2 构建 APP 镜像

```bash
# 1. 克隆项目（如果还没有）
cd /opt/weknora
git clone https://github.com/Tencent/WeKnora.git .
# 或使用已有代码
cd /opt/weknora/WeKnora

# 2. 使用构建脚本构建 APP 镜像
./scripts/build_images.sh --app

# 或使用 Makefile
make build-images-app

# 3. 验证镜像
docker images | grep weknora-app
```

**构建参数说明**：
- `--platform linux/amd64`：指定平台（根据服务器架构调整）
- 构建时间：约 5-10 分钟（取决于网络和 CPU）

### 2.3 构建 FRONTEND 镜像

```bash
# 1. 使用构建脚本构建 FRONTEND 镜像
./scripts/build_images.sh --frontend

# 或使用 Makefile
make build-images-frontend

# 2. 验证镜像
docker images | grep weknora-ui
```

**构建参数说明**：
- 构建时间：约 3-5 分钟（取决于网络和 CPU）
- 需要 Node.js 环境（已在 Dockerfile 中处理）

### 2.4 验证本地镜像

```bash
# 查看所有构建的镜像
docker images | grep -E "weknora-app|weknora-ui"

# 应该看到类似输出：
# wechatopenai/weknora-app        latest    xxx    xxx ago    xxx MB
# wechatopenai/weknora-ui         latest    xxx    xxx ago    xxx MB
```

## 四、部署步骤

### 4.1 完整部署流程

#### 步骤 1：准备环境

```bash
# 1. 登录服务器
ssh user@test-server

# 2. 创建部署目录
sudo mkdir -p /opt/weknora
sudo chown $USER:$USER /opt/weknora
cd /opt/weknora

# 3. 克隆项目代码
git clone https://github.com/Tencent/WeKnora.git .
# 或上传代码包并解压
```

#### 步骤 2：构建本地镜像

```bash
# 1. 进入项目目录
cd /opt/weknora

# 2. 构建 APP 镜像（约 5-10 分钟）
./scripts/build_images.sh --app

# 3. 构建 FRONTEND 镜像（约 3-5 分钟）
./scripts/build_images.sh --frontend

# 4. 验证镜像
docker images | grep -E "weknora-app|weknora-ui"
```

#### 步骤 3：准备配置文件

```bash
# 1. 创建数据目录
sudo mkdir -p /data/weknora/{postgres,redis,minio,neo4j,files,logs}
sudo chown -R $USER:$USER /data/weknora

# 2. 复制配置文件
cp docker-compose.test.yml docker-compose.yml
cp .env.test .env

# 3. 编辑 .env 文件，修改密码和密钥
vim .env
```

#### 步骤 4：拉取开源镜像

```bash
# 拉取所有需要的开源镜像
docker pull paradedb/paradedb:v0.21.4-pg17
docker pull redis:7.0-alpine
docker pull minio/minio:RELEASE.2025-09-07T16-13-09Z
docker pull neo4j:2025.10.1
docker pull wechatopenai/weknora-docreader:latest

# 验证镜像
docker images
```

#### 步骤 5：启动服务

```bash
# 1. 启动所有服务（按依赖顺序）
docker compose up -d

# 2. 查看服务状态
docker compose ps

# 3. 查看日志
docker compose logs -f
```

#### 步骤 6：初始化 MinIO Bucket

```bash
# 1. 等待 MinIO 启动完成
docker compose logs minio

# 2. 访问 MinIO 控制台
# http://服务器IP:9001
# 用户名：minioadmin
# 密码：minioadmin

# 3. 创建 Bucket：weknora-files
# 或在容器内执行
docker compose exec minio mc alias set myminio http://localhost:9000 minioadmin minioadmin
docker compose exec minio mc mb myminio/weknora-files
docker compose exec minio mc anonymous set download myminio/weknora-files
```

#### 步骤 7：执行数据库迁移

```bash
# 1. 等待 PostgreSQL 启动完成
docker compose logs postgres

# 2. 执行数据库迁移
docker compose exec app ./scripts/migrate.sh up

# 或手动执行
docker compose exec app migrate -path ./migrations/versioned -database "postgres://weknora:weknora123456@postgres:5432/weknora?sslmode=disable" up
```

#### 步骤 8：验证服务

```bash
# 1. 检查服务健康状态
curl http://localhost:8080/health

# 2. 检查前端
curl http://localhost:80

# 3. 检查所有容器状态
docker compose ps
```

## 六、资源分配

### 6.1 内存分配（16GB 总内存）

| 服务 | 限制内存 | 预留内存 | 说明 |
|------|---------|---------|------|
| PostgreSQL | 4GB | 2GB | 数据库服务，内存需求较大 |
| APP | 2GB | 1GB | 后端应用服务 |
| Neo4j | 2GB | 1GB | 知识图谱服务 |
| Docreader | 1GB | 512MB | 文档解析服务 |
| MinIO | 1GB | 512MB | 对象存储服务 |
| Redis | 512MB | 256MB | 缓存服务 |
| Frontend | 512MB | 256MB | 前端服务 |
| **系统预留** | **5GB** | - | 操作系统和其他进程 |
| **总计** | **16GB** | **5.5GB** | - |

### 6.2 CPU 分配（8 核）

| 服务 | CPU 限制 | 说明 |
|------|---------|------|
| PostgreSQL | 2 核 | 数据库查询和索引 |
| APP | 2 核 | 业务逻辑处理 |
| Docreader | 1 核 | 文档解析（CPU 密集型） |
| Neo4j | 1 核 | 图数据库查询 |
| 其他服务 | 共享剩余 | MinIO、Redis、Frontend |
| **系统预留** | **1 核** | 操作系统 |

### 6.3 存储分配（100GB）

| 用途 | 预估大小 | 说明 |
|------|---------|------|
| PostgreSQL 数据 | 20GB | 数据库数据和索引 |
| MinIO 对象存储 | 50GB | 用户上传的文件 |
| Neo4j 数据 | 10GB | 知识图谱数据 |
| Redis 数据 | 1GB | 缓存数据（AOF） |
| 日志文件 | 5GB | 应用日志 |
| 系统文件 | 10GB | 操作系统和 Docker |
| **预留空间** | **4GB** | 临时文件和备份 |

## 七、运维管理

### 7.1 日常运维命令

```bash
# 1. 启动服务
docker compose up -d

# 2. 停止服务
docker compose stop

# 3. 重启服务
docker compose restart

# 4. 重启特定服务
docker compose restart app

# 5. 查看服务状态
docker compose ps

# 6. 查看资源使用
docker stats

# 7. 查看磁盘使用
docker system df
df -h /data/weknora
```

### 7.2 更新部署

```bash
# 1. 更新代码
cd /opt/weknora
git pull

# 2. 重新构建镜像（仅 APP 和 FRONTEND）
./scripts/build_images.sh --app
./scripts/build_images.sh --frontend

# 3. 停止服务
docker compose stop app frontend

# 4. 重新启动服务
docker compose up -d app frontend

# 5. 验证更新
docker compose logs -f app
```

### 7.3 备份与恢复

#### 备份数据库

```bash
# 1. 备份 PostgreSQL
docker compose exec postgres pg_dump -U weknora weknora > /data/weknora/backup/postgres_$(date +%Y%m%d_%H%M%S).sql

# 2. 备份 Neo4j
docker compose exec neo4j neo4j-admin database dump --database=neo4j --to-path=/data
docker compose cp neo4j:/data /data/weknora/backup/neo4j_$(date +%Y%m%d_%H%M%S)

# 3. 备份 MinIO 数据
docker compose exec minio mc mirror myminio/weknora-files /data/backup/minio_$(date +%Y%m%d_%H%M%S)
```

#### 恢复数据库

```bash
# 1. 恢复 PostgreSQL
docker compose exec -T postgres psql -U weknora weknora < /data/weknora/backup/postgres_20250128_120000.sql

# 2. 恢复 Neo4j
docker compose exec neo4j neo4j-admin database load --database=neo4j --from-path=/data/backup/neo4j_20250128_120000
```

## 八、常见问题

### 8.1 镜像构建失败

**问题**：构建 APP 或 FRONTEND 镜像时失败

**解决方案**：

```bash
# 1. 检查网络连接
ping -c 3 8.8.8.8

# 2. 检查 Docker 服务
sudo systemctl status docker

# 3. 清理 Docker 缓存
docker system prune -a

# 4. 重新构建（带详细日志）
docker build --no-cache -f docker/Dockerfile.app -t wechatopenai/weknora-app:latest .
```

### 8.2 服务启动失败

**问题**：容器启动后立即退出

**解决方案**：

```bash
# 1. 查看容器日志
docker compose logs app

# 2. 检查环境变量
docker compose config

# 3. 检查数据目录权限
ls -la /data/weknora

# 4. 检查端口占用
netstat -tulpn | grep -E "8080|80|5432|6379|9000"
```

### 8.3 数据库连接失败

**问题**：APP 服务无法连接 PostgreSQL

**解决方案**：

```bash
# 1. 检查 PostgreSQL 是否运行
docker compose ps postgres

# 2. 检查数据库日志
docker compose logs postgres

# 3. 测试数据库连接
docker compose exec postgres psql -U weknora -d weknora -c "SELECT 1;"

# 4. 检查环境变量
docker compose exec app env | grep DB_
```

### 8.4 MinIO Bucket 不存在

**问题**：文件上传失败，提示 Bucket 不存在

**解决方案**：

```bash
# 1. 创建 Bucket
docker compose exec minio mc alias set myminio http://localhost:9000 minioadmin minioadmin
docker compose exec minio mc mb myminio/weknora-files
docker compose exec minio mc anonymous set download myminio/weknora-files

# 2. 验证 Bucket
docker compose exec minio mc ls myminio/
```

### 8.5 内存不足

**问题**：容器因内存不足被 OOM Killer 杀死

**解决方案**：

```bash
# 1. 检查内存使用
free -h
docker stats --no-stream

# 2. 调整内存限制（在 docker-compose.yml 中）
# 减少某些服务的 memory limits

# 3. 增加交换空间（临时方案）
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

## 十、快速部署脚本

创建 `scripts/deploy_test.sh`：

```bash
#!/bin/bash
# WeKnora 测试环境一键部署脚本

set -e

echo "=========================================="
echo "WeKnora 测试环境部署脚本"
echo "=========================================="

docker --version || { echo "Docker 未安装"; exit 1; }
docker compose version || { echo "Docker Compose 未安装"; exit 1; }

# 2. 创建数据目录
echo "[2/8] 创建数据目录..."
sudo mkdir -p /data/weknora/{postgres,redis,minio,neo4j,files,logs,backup}
sudo chown -R $USER:$USER /data/weknora

# 3. 构建 APP 镜像
echo "[3/8] 构建 APP 镜像..."
./scripts/build_images.sh --app

# 4. 构建 FRONTEND 镜像
echo "[4/8] 构建 FRONTEND 镜像..."
./scripts/build_images.sh --frontend

# 5. 拉取开源镜像
echo "[5/8] 拉取开源镜像..."
docker pull paradedb/paradedb:v0.21.4-pg17
docker pull redis:7.0-alpine
docker pull minio/minio:RELEASE.2025-09-07T16-13-09Z
docker pull neo4j:2025.10.1
docker pull wechatopenai/weknora-docreader:latest

# 6. 启动服务
echo "[6/8] 启动服务..."
docker compose -f docker-compose.test.yml up -d

# 7. 等待服务就绪
echo "[7/8] 等待服务就绪..."
sleep 30

# 8. 初始化 MinIO Bucket
echo "[8/8] 初始化 MinIO Bucket..."
docker compose exec -T minio sh -c 'mc alias set myminio http://localhost:9000 minioadmin minioadmin && mc mb myminio/weknora-files && mc anonymous set download myminio/weknora-files' || echo "MinIO Bucket 初始化失败，请手动创建"

# 9. 执行数据库迁移

echo "=========================================="
echo "部署完成！"
echo "=========================================="
echo "前端地址: http://$(hostname -I | awk '{print $1}')"
echo "API地址: http://$(hostname -I | awk '{print $1}'):8080"
echo "MinIO控制台: http://$(hostname -I | awk '{print $1}'):9001"
echo "Neo4j浏览器: http://$(hostname -I | awk '{print $1}'):7474"
echo ""
echo "查看服务状态: docker compose ps"
echo "查看日志: docker compose logs -f"
```
