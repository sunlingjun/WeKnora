## 基础信息

- **基础 URL**: `http://localhost:8080/api/v1`
- **API 版本**: v1
- **响应格式**: JSON
- **认证方式**: API Key (通过 `X-API-Key` 请求头)
- **Swagger 文档**: `http://localhost:8080/swagger/index.html`

---

## 认证机制

所有 API 请求需要在 HTTP 请求头中包含 `X-API-Key` 进行身份认证：

```http
X-API-Key: your_api_key
```

为便于问题追踪和调试，建议每个请求的 HTTP 请求头中添加 `X-Request-ID`：

```http
X-Request-ID: unique_request_id
```

### 获取 API Key

在 Web 页面完成账户注册后，前往账户信息页面获取您的 API Key。

### 2. 知识库管理

知识库是 WeKnora 的核心概念，用于组织和存储知识内容。

#### 2.1 创建知识库

```http
POST /api/v1/knowledge-bases
Content-Type: application/json
X-API-Key: your_api_key

{
  "name": "知识库名称",
  "description": "知识库描述",
  "chunking_config": {
    "chunk_size": 1000,
    "chunk_overlap": 200,
    "separators": ["\n\n", "\n", "。", "！", "？"],
    "enable_multimodal": true
  },
  "embedding_model_id": "embedding_model_id",
  "summary_model_id": "summary_model_id",
  "rerank_model_id": "rerank_model_id"
}
```

#### 2.2 获取知识库列表

```http
GET /api/v1/knowledge-bases
X-API-Key: your_api_key
```

#### 2.3 获取知识库详情

```http
GET /api/v1/knowledge-bases/{id}
X-API-Key: your_api_key
```

#### 2.4 更新知识库

```http
PUT /api/v1/knowledge-bases/{id}
Content-Type: application/json
X-API-Key: your_api_key

{
  "name": "更新后的名称",
  "description": "更新后的描述"
}
```

#### 2.5 删除知识库

```http
DELETE /api/v1/knowledge-bases/{id}
X-API-Key: your_api_key
```

#### 2.6 混合搜索

```http
GET /api/v1/knowledge-bases/{id}/hybrid-search
Content-Type: application/json
X-API-Key: your_api_key

{
  "query_text": "搜索关键词",
  "vector_threshold": 0.5,
  "match_count": 10
}
```

**完整接口列表**:

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/knowledge-bases` | 创建知识库 |
| GET | `/knowledge-bases` | 获取知识库列表 |
| GET | `/knowledge-bases/:id` | 获取知识库详情 |
| PUT | `/knowledge-bases/:id` | 更新知识库 |
| DELETE | `/knowledge-bases/:id` | 删除知识库 |
| POST | `/knowledge-bases/copy` | 拷贝知识库 |
| GET | `/knowledge-bases/:id/hybrid-search` | 混合搜索（向量+关键词） |

**详细文档**: [知识库管理 API](./api/knowledge-base.md)

### 3. 知识管理

知识是存储在知识库中的具体内容，可以是文件、URL 或手工录入的 Markdown。

#### 3.1 从文件创建知识

```http
POST /api/v1/knowledge-bases/{id}/knowledge/file
Content-Type: multipart/form-data
X-API-Key: your_api_key

file: [文件]
enable_multimodel: true
metadata: {"key": "value"}
```

#### 3.2 从 URL 创建知识

```http
POST /api/v1/knowledge-bases/{id}/knowledge/url
Content-Type: application/json
X-API-Key: your_api_key

{
  "url": "https://example.com/article",
  "enable_multimodel": true
}
```

#### 3.3 创建手工 Markdown 知识

```http
POST /api/v1/knowledge-bases/{id}/knowledge/manual
Content-Type: application/json
X-API-Key: your_api_key

{
  "title": "标题",
  "content": "# Markdown 内容"
}
```

#### 3.4 获取知识列表

```http
GET /api/v1/knowledge-bases/{id}/knowledge?page=1&page_size=20&tag_id=tag_id
X-API-Key: your_api_key
```

#### 3.5 获取知识详情

```http
GET /api/v1/knowledge/{id}
X-API-Key: your_api_key
```

#### 3.6 删除知识

```http
DELETE /api/v1/knowledge/{id}
X-API-Key: your_api_key
```

**完整接口列表**:

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/knowledge-bases/:id/knowledge/file` | 从文件创建知识 |
| POST | `/knowledge-bases/:id/knowledge/url` | 从 URL 创建知识 |
| POST | `/knowledge-bases/:id/knowledge/manual` | 创建手工 Markdown 知识 |
| GET | `/knowledge-bases/:id/knowledge` | 获取知识库下的知识列表 |
| GET | `/knowledge/:id` | 获取知识详情 |
| GET | `/knowledge/batch` | 批量获取知识 |
| DELETE | `/knowledge/:id` | 删除知识 |
| PUT | `/knowledge/:id` | 更新知识 |
| PUT | `/knowledge/manual/:id` | 更新手工 Markdown 知识 |
| GET | `/knowledge/:id/download` | 下载知识文件 |

**详细文档**: [知识管理 API](./api/knowledge.md)

### 5. 聊天服务

聊天服务提供基于知识库的问答和 Agent 模式的智能问答。

#### 5.1 基于知识库的问答（流式）

```http
POST /api/v1/knowledge-chat/{session_id}
Content-Type: application/json
X-API-Key: your_api_key

{
  "query": "用户问题"
}
```

**响应格式**: Server-Sent Events (SSE)

**响应类型**:
- `references`: 知识库检索引用
- `answer`: 回答内容（流式）

#### 5.2 基于 Agent 的智能问答（流式）

```http
POST /api/v1/agent-chat/{session_id}
Content-Type: application/json
X-API-Key: your_api_key

{
  "query": "用户问题",
  "agent_enabled": true,
  "web_search_enabled": true,
  "knowledge_base_ids": ["kb-00000001"]
}
```

**响应格式**: Server-Sent Events (SSE)

**响应类型**:
- `thinking`: Agent 思考过程
- `tool_call`: 工具调用信息
- `tool_result`: 工具调用结果
- `references`: 知识库检索引用
- `answer`: 最终回答内容
- `reflection`: Agent 反思内容
- `error`: 错误信息

#### 5.3 知识搜索

```http
POST /api/v1/knowledge-search
Content-Type: application/json
X-API-Key: your_api_key

{
  "query": "搜索关键词",
  "knowledge_base_ids": ["kb-00000001"]
}
```

**注意**: Agent 模式下，如果知识库检索结果不足，会自动调用联网搜索补充信息。详见 [联网搜索配置指南](./联网搜索配置指南.md)。

**完整接口列表**:

| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/knowledge-chat/:session_id` | 基于知识库的问答（流式） |
| POST | `/agent-chat/:session_id` | 基于 Agent 的智能问答（流式） |
| POST | `/knowledge-search` | 基于知识库的搜索知识 |
| GET | `/web-search/providers` | 获取可用的搜索引擎提供商列表 |

**详细文档**: 
- [聊天功能 API](./api/chat.md)
- [联网搜索配置指南](./联网搜索配置指南.md)

## 错误处理

所有 API 使用标准的 HTTP 状态码表示请求状态，并返回统一的错误响应格式：

```json
{
  "success": false,
  "error": {
    "code": "错误代码",
    "message": "错误信息",
    "details": "错误详情"
  }
}
```

### 常见 HTTP 状态码

- `200 OK`: 请求成功
- `201 Created`: 创建成功
- `400 Bad Request`: 请求参数错误
- `401 Unauthorized`: 未授权（API Key 无效）
- `403 Forbidden`: 禁止访问
- `404 Not Found`: 资源不存在
- `500 Internal Server Error`: 服务器内部错误

## 流式响应说明

聊天接口（`/knowledge-chat` 和 `/agent-chat`）使用 Server-Sent Events (SSE) 格式返回流式响应。

### 响应格式

```
event: message
data: {"id":"...","response_type":"answer","content":"回答内容","done":false}

event: message
data: {"id":"...","response_type":"answer","content":"","done":true}
```

### 客户端处理示例

```javascript
const eventSource = new EventSource('/api/v1/knowledge-chat/session_id', {
  headers: {
    'X-API-Key': 'your_api_key'
  }
});

eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.response_type === 'answer') {
    // 处理回答内容
    console.log(data.content);
  }
  if (data.done) {
    eventSource.close();
  }
};
```
