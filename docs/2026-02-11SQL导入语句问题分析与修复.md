### 2.2 改进建议

1. **添加类型转换**：为 JSONB 和 TIMESTAMP 字段添加显式类型转换
   ```sql
   '{"engines": []}'::jsonb
   '2026-01-19 13:47:16.963303+08'::timestamptz
   ```

2. **使用 ON CONFLICT**：避免重复插入
   ```sql
   INSERT INTO public.users (...)
   VALUES (...)
   ON CONFLICT (id) DO NOTHING;  -- 如果已存在则跳过
   ```

3. **使用事务**：确保数据一致性
   ```sql
   BEGIN;
   INSERT INTO public.tenants (...) VALUES (...);
   INSERT INTO public.users (...) VALUES (...);
   COMMIT;
   ```

## 五、常见问题

### Q1: 导入后密码验证失败，但哈希值看起来正确？

**可能原因**：
1. 密码哈希值在导入时被截断（检查长度是否为 60）
2. 导入时添加了空格或换行符
3. 字符编码问题

**解决方法**：
```sql
-- 检查并清理
SELECT 
    email,
    LENGTH(password_hash) as length,
    password_hash != TRIM(password_hash) as has_whitespace
FROM users
WHERE email = 'slj0713@163.com';

-- 如果有空格，清理
UPDATE users 
SET password_hash = TRIM(password_hash)
WHERE email = 'slj0713@163.com';
```

### Q2: tenants 表的单引号转义问题

**问题**：`'slj001's Workspace'` 会报错

**解决**：使用两个单引号转义
```sql
'slj001''s Workspace'  -- ✅ 正确
'slj001's Workspace'   -- ❌ 错误
```

### Q3: 如何批量导入多个用户？

**方法**：使用事务和循环
```sql
BEGIN;

DO $$
DECLARE
    user_record RECORD;
BEGIN
    FOR user_record IN 
        SELECT * FROM source_users_table
    LOOP
        INSERT INTO public.users (...)
        VALUES (...)
        ON CONFLICT (id) DO NOTHING;
    END LOOP;
END $$;

COMMIT;
```
