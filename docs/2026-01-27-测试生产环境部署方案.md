## 一、环境概述

### 1.1 环境分类

| 环境类型 | 用途 | 部署方式 | 高可用 | 数据持久化 |
|---------|------|---------|--------|-----------|
| **开发环境** | 本地开发调试 | Docker Compose | ❌ | ✅ |
| **测试环境** | 功能测试、集成测试 | Docker Compose | ⚠️ 可选 | ✅ |
| **生产环境** | 正式服务 | Docker Compose | ✅（多机） | ✅ |

### 1.2 系统架构

```
                    ┌─────────────┐
                    │   Nginx     │
                    │  (反向代理)  │
                    └──────┬──────┘
                           │
           ┌───────────────┴───────────────┐
           │                               │
           ▼                               ▼
    ┌─────────────┐                 ┌─────────────┐
    │  Frontend   │                 │   Backend   │
    │  (Vue.js)   │                 │   (Go/Gin)  │
    │  Nginx      │                 │   API       │
    └─────────────┘                 └──────┬──────┘
                                           │
                    ┌──────────────────────┼──────────────────────┐
                    │                      │                      │
                    ▼                      ▼                      ▼
             ┌─────────────┐        ┌─────────────┐        ┌─────────────┐
             │  Docreader  │        │  PostgreSQL │        │    Redis    │
             │   (gRPC)   │        │  (ParadeDB) │        │   (Queue)   │
             └─────────────┘        └─────────────┘        └─────────────┘
                                           │
                    ┌──────────────────────┼──────────────────────┐
                    │                      │                      │
                    ▼                      ▼                      ▼
             ┌─────────────┐        ┌─────────────┐        ┌─────────────┐
             │   MinIO     │        │   Neo4j     │        │   (其他)    │
             │ 对象存储    │        │ 知识图谱    │        │             │
             └─────────────┘        └─────────────┘        └─────────────┘
```

### 1.3 核心组件

| 组件 | 说明 | 必需性 |
|------|------|--------|
| **Frontend** | Vue.js 前端界面 | ✅ 必需 |
| **Backend (App)** | Go/Gin 后端 API | ✅ 必需 |
| **Docreader** | Python gRPC 文档解析服务 | ✅ 必需 |
| **PostgreSQL** | ParadeDB（向量+全文检索） | ✅ 必需 |
| **Redis** | 任务队列和缓存 | ✅ 必需 |
| **MinIO** | 对象存储（文件存储） | ✅ **必需**（本方案） |
| **Neo4j** | 知识图谱（GraphRAG） | ⚠️ 可选（启用 GraphRAG 时必需） |
| **Qdrant** | 向量数据库（可选） | ⚠️ 可选 |
| **Jaeger** | 链路追踪（可选） | ⚠️ 可选 |

## 二、测试环境部署方案

### 2.1 部署方式

**推荐方案**：Docker Compose（单机部署）

### 2.2 服务器配置

#### 方案一：单机部署（推荐）

| 资源项 | 配置 | 说明 |
|--------|------|------|
| **服务器数量** | 1 台 | 单机部署所有服务 |
| **CPU** | 4 核 | 满足基本测试需求 |
| **内存** | 8GB | 最低配置 |
| **磁盘** | 100GB SSD | 数据库+MinIO存储 |
| **网络** | 内网 | 测试环境通常在内网 |

**资源分配**：
- PostgreSQL: 2GB 内存
- Redis: 512MB 内存
- App: 1GB 内存
- Docreader: 512MB 内存
- Frontend: 256MB 内存
- MinIO: 1GB 内存
- 其他服务: 剩余内存

### 2.5 部署步骤

```bash
# 1. 克隆代码
git clone https://github.com/Tencent/WeKnora.git
cd WeKnora

# 2. 配置环境变量
cp .env.example .env.test
# 编辑 .env.test，填入测试环境配置

# 3. 启动所有服务（包括 MinIO）
docker compose -f docker-compose.test.yml --env-file .env.test up -d

# 4. 等待服务启动（约30-60秒）
sleep 60

# 5. 创建 MinIO 存储桶
# 方式一：通过 MinIO Console（推荐）
# 访问 http://localhost:9001，使用 minioadmin/minioadmin 登录
# 创建存储桶：weknora-test

# 方式二：使用 MinIO Client
docker run --rm -it --network weknora-slj_WeKnora-network \
  minio/mc alias set local http://minio:9000 minioadmin minioadmin
docker run --rm -it --network weknora-slj_WeKnora-network \
  minio/mc mb local/weknora-test

# 6. 执行数据库迁移
docker compose -f docker-compose.test.yml exec app ./migrate up

# 7. 验证服务状态
curl http://localhost:8080/health
curl http://localhost:8080/api/v1/system/info  # 检查 minio_enabled

# 8. 查看日志
docker compose -f docker-compose.test.yml logs -f
```

## 三、生产环境部署方案

### 3.1 部署方式

**推荐方案**：Docker Compose（多机部署，可选）

**单机方案**：适用于小型生产环境（< 100 用户）

**多机方案**：适用于中大型生产环境（> 100 用户）

### 3.2 服务器配置方案

#### 方案一：单机部署（小型生产环境）

| 资源项 | 配置 | 说明 |
|--------|------|------|
| **服务器数量** | 1 台 | 单机部署所有服务 |
| **CPU** | 8 核 | 满足小型生产需求 |
| **内存** | 16GB | 推荐配置 |
| **磁盘** | 500GB SSD | 数据库+MinIO存储 |
| **网络** | 公网 IP | 公网访问 |

**资源分配**：
- PostgreSQL: 4GB 内存
- Redis: 1GB 内存
- App: 2GB 内存
- Docreader: 1GB 内存
- Frontend: 512MB 内存
- MinIO: 2GB 内存
- Neo4j: 2GB 内存（如启用）
- 系统预留: 2GB

#### 方案二：多机部署（中大型生产环境）

| 资源项 | 配置 | 说明 |
|--------|------|------|
| **服务器数量** | 3-5 台 | 应用服务器 + 数据库服务器 + MinIO服务器 |
| **应用服务器** | 8 核 / 16GB × 2 | 运行 App、Frontend、Docreader |
| **数据库服务器** | 8 核 / 16GB × 1 | 运行 PostgreSQL、Redis |
| **MinIO服务器** | 4 核 / 8GB × 1 | 运行 MinIO（可选，也可与应用服务器合并） |
| **磁盘** | 每台 500GB SSD | 分布式存储 |
| **网络** | 公网 IP + 内网 | 公网访问，内网通信 |

**资源分配**：
- PostgreSQL: 8GB 内存（专用节点）
- Redis: 2GB 内存
- App: 4GB 内存 × 2 = 8GB
- Docreader: 2GB 内存 × 2 = 4GB
- Frontend: 512MB 内存 × 2 = 1GB
- MinIO: 4GB 内存（专用节点或与应用服务器合并）
- Neo4j: 4GB 内存（如启用，可单独节点或与应用服务器合并）
- 系统预留: 2GB

## 五、Neo4j 配置与启用

### 5.1 Neo4j 概述

Neo4j 是 WeKnora 的知识图谱数据库，用于存储实体和关系数据，支持 GraphRAG（图增强检索）功能。

**功能说明**：
- ✅ **实体抽取**：从文档中提取实体（人物、地点、概念等）
- ✅ **关系抽取**：提取实体之间的关系
- ✅ **知识图谱构建**：自动构建知识图谱
- ✅ **GraphRAG**：在对话时查询知识图谱，增强检索效果

**使用场景**：
- 需要理解文档中的实体和关系
- 需要基于知识图谱进行智能问答
- 需要可视化知识图谱

### 5.2 Neo4j 启用步骤

### 5.6 Neo4j 常见问题

#### 问题 1：无法连接 Neo4j - `dial tcp: lookup neo4j: no such host`

**错误信息**：
```
Failed to verify Neo4j authentication: ConnectivityError: 
Unable to retrieve routing table from neo4j:7687: dial tcp: lookup neo4j: no such host
```

**原因分析**：
- **Neo4j 容器未启动**：Neo4j 服务使用了 `profiles`，默认不会启动
- **环境变量配置错误**：设置了 `NEO4J_ENABLE=true`，但 Neo4j 容器未启动
- **启动顺序问题**：应用容器在 Neo4j 容器启动之前就尝试连接

**排查步骤**：
```bash
# 1. 检查 Neo4j 容器状态
docker compose ps neo4j

# 2. 检查环境变量
docker compose exec app env | grep NEO4J

# 3. 检查网络连接
docker compose exec app ping -c 2 neo4j

# 4. 检查端口连接
docker compose exec app nc -zv neo4j 7687
```

**解决方案**：

**方案 A：不需要 Neo4j（推荐）**
```bash
# 禁用 Neo4j
# 修改 .env 文件
NEO4J_ENABLE=false

# 重启应用
docker compose restart app
```

**方案 B：需要 Neo4j**
```bash
# 1. 启动 Neo4j 容器
docker compose --profile neo4j up -d neo4j

# 2. 等待 Neo4j 启动（约60秒）
sleep 60

# 3. 验证 Neo4j 启动
docker compose ps neo4j
curl http://localhost:7474

# 4. 重启应用
docker compose restart app
```

**方案 C：修复 Docker Compose 配置**
```yaml
# 在 docker-compose.yml 的 app 服务中添加 depends_on
app:
  depends_on:
    # ... 其他依赖 ...
    neo4j:  # 添加 Neo4j 依赖
      condition: service_started
```

**详细排查指南**：请参考 [Neo4j 连接问题排查](./Neo4j连接问题排查.md)

#### 问题 2：未生成节点

**排查步骤**：
```bash
# 1. 检查知识库是否启用实体/关系抽取
# 2. 查看后端日志
docker compose logs app | grep -i "entity\|relation"

# 3. 检查 Neo4j 中是否有数据
# 访问 Neo4j Browser，执行：MATCH (n) RETURN count(n);
```

**解决方案**：
- 确保知识库已启用实体抽取和关系抽取
- 确保已上传文档并完成解析
- 确保 LLM 模型配置正确
- 查看后端日志中的错误信息

#### 问题 3：GraphRAG 未生效

**排查步骤**：
```bash
# 1. 检查 ENABLE_GRAPH_RAG 环境变量
docker compose exec app env | grep ENABLE_GRAPH_RAG

# 2. 检查 Neo4j 连接
docker compose logs app | grep -i "graphrag\|neo4j"
```

**解决方案**：

- 确保知识图谱中有数据

## 十、备份与恢复

### 9.1 数据库备份

#### 自动备份脚本

```bash
#!/bin/bash
# backup-postgres.sh

BACKUP_DIR="/backup/postgres"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/weknora_${DATE}.sql"

# 创建备份目录
mkdir -p ${BACKUP_DIR}

# 执行备份
docker compose exec -T postgres pg_dump -U ${DB_USER} ${DB_NAME} > ${BACKUP_FILE}

# 压缩备份
gzip ${BACKUP_FILE}

# 删除 7 天前的备份
find ${BACKUP_DIR} -name "*.sql.gz" -mtime +7 -delete

echo "Backup completed: ${BACKUP_FILE}.gz"
```

#### Cron 定时任务

```bash
# 每天凌晨 2 点执行备份
0 2 * * * /path/to/backup-postgres.sh
```

### 10.2 MinIO 备份

#### 使用 MinIO Client 备份

```bash
#!/bin/bash
# backup-minio.sh

BACKUP_DIR="/backup/minio"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_BUCKET="weknora-backup-${DATE}"

# 配置 MinIO Client
mc alias set local http://localhost:9000 ${MINIO_ACCESS_KEY_ID} ${MINIO_SECRET_ACCESS_KEY}

# 创建备份存储桶
mc mb local/${BACKUP_BUCKET}

# 复制数据
mc mirror local/weknora local/${BACKUP_BUCKET}

echo "MinIO backup completed: ${BACKUP_BUCKET}"
```

### 10.3 Neo4j 备份

#### 使用 Neo4j 导出工具备份

```bash
#!/bin/bash
# backup-neo4j.sh

BACKUP_DIR="/backup/neo4j"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/weknora_${DATE}.dump"

# 创建备份目录
mkdir -p ${BACKUP_DIR}

# 执行备份（使用 Neo4j Admin）
docker compose exec neo4j neo4j-admin database dump neo4j --to-path=/backup
docker compose cp neo4j:/backup/neo4j.dump ${BACKUP_FILE}

# 压缩备份
gzip ${BACKUP_FILE}

# 删除 7 天前的备份
find ${BACKUP_DIR} -name "*.dump.gz" -mtime +7 -delete

echo "Neo4j backup completed: ${BACKUP_FILE}.gz"
```

### 10.4 恢复流程

#### 数据库恢复

```bash
# 1. 停止应用
docker compose stop app

# 2. 恢复数据库
docker compose exec -T postgres psql -U ${DB_USER} ${DB_NAME} < backup.sql

# 3. 启动应用
docker compose start app
```

#### MinIO 恢复

```bash
# 使用 MinIO Client 恢复
mc mirror local/weknora-backup-20250127 local/weknora
```

#### Neo4j 恢复

```bash
# 1. 停止应用
docker compose stop app

# 2. 停止 Neo4j
docker compose stop neo4j

# 3. 恢复数据库
docker compose exec neo4j neo4j-admin database load neo4j --from-path=/backup/neo4j.dump --overwrite-destination=true

# 4. 启动 Neo4j
docker compose start neo4j

# 5. 启动应用
docker compose start app
```

## 十二、常见问题

### 11.1 MinIO 连接失败

**问题**：应用启动时报错 "missing MinIO configuration" 或连接失败

**排查步骤**：
```bash
# 1. 检查 MinIO 容器状态
docker compose ps minio

# 2. 检查 MinIO 日志
docker compose logs minio

# 3. 检查环境变量
docker compose exec app env | grep MINIO

# 4. 测试 MinIO 连接
curl http://localhost:9000/minio/health/live

# 5. 检查存储桶是否存在
docker compose exec minio mc ls local/
```

**解决方案**：
- 确保所有 MinIO 环境变量已设置
- 确保 MinIO 容器已启动
- 确保存储桶已创建
- 检查网络连接（Docker Compose 中使用服务名 `minio:9000`）

### 11.2 存储桶不存在

**问题**：文件上传失败，提示存储桶不存在

**解决方案**：
```bash
# 1. 登录 MinIO Console
# 访问 http://localhost:9001

# 2. 创建存储桶（名称与 MINIO_BUCKET_NAME 一致）

# 3. 或使用 MinIO Client
mc mb local/weknora
```

### 11.3 容器无法启动

**问题**：容器状态为 `Exited` 或 `Restarting`

**排查步骤**：
```bash
# 1. 查看容器日志
docker compose logs <service-name>

# 2. 查看容器状态
docker compose ps

# 3. 检查资源限制
docker stats

# 4. 检查磁盘空间
df -h
```

### 11.4 性能问题

**问题**：API 响应慢或超时

**排查步骤**：
```bash
# 1. 查看资源使用
docker stats

# 2. 查看应用日志
docker compose logs app --tail=100

# 3. 检查数据库性能
docker compose exec postgres psql -U weknora -c "SELECT * FROM pg_stat_statements ORDER BY total_time DESC LIMIT 10;"

# 4. 检查 MinIO 性能
docker compose logs minio --tail=100

# 5. 检查 Neo4j 性能（如启用）
docker compose logs neo4j --tail=100
docker compose exec neo4j cypher-shell -u neo4j -p ${NEO4J_PASSWORD} "CALL dbms.queryJmx('org.neo4j:instance=kernel#0,name=Page cache') YIELD attributes RETURN attributes"
```
