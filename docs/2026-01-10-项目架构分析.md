## 🏗️ 核心组件详解

### 1. 前端应用 (Frontend)

**技术栈：**
- Vue 3.5+ (Composition API)
- TypeScript
- TDesign Vue Next (UI组件库)
- Vite (构建工具)
- Pinia (状态管理)
- Vue Router (路由)
- Axios (HTTP客户端)

**主要功能模块：**
- 知识库管理（创建、编辑、导入文档）
- 对话界面（Agent模式/普通模式）
- 会话管理
- 模型配置
- 用户认证

**目录结构：**
```
frontend/
├── src/
│   ├── api/          # API接口定义
│   ├── components/   # 组件
│   ├── views/        # 页面视图
│   ├── stores/        # Pinia状态管理
│   ├── router/        # 路由配置
│   ├── utils/         # 工具函数
│   └── i18n/          # 国际化
├── public/            # 静态资源
└── package.json       # 依赖配置
```

**部署方式：**
- 开发模式：`npm run dev` (Vite开发服务器)
- 生产模式：构建为静态文件，由Nginx提供服务

---

### 2. 后端应用服务 (App Service)

**技术栈：**
- Go 1.24+
- Gin (Web框架)
- GORM (ORM)
- Uber Dig (依赖注入)
- Swagger (API文档)

**核心模块：**

#### 2.1 Agent引擎 (`internal/agent/`)
- ReACT Agent实现
- 工具调用管理
- MCP工具集成
- Web搜索集成

#### 2.2 知识库服务 (`internal/application/service/`)
- 知识库CRUD
- 文档导入和处理
- 向量索引管理
- 检索服务

#### 2.3 会话管理 (`internal/application/service/`)
- 多轮对话管理
- 上下文维护
- 会话历史存储

#### 2.4 模型管理 (`internal/application/service/`)
- LLM模型配置
- Embedding模型管理
- Rerank模型管理
- 模型调用封装

#### 2.5 检索引擎 (`internal/searchutil/`)
- 混合检索（BM25 + 向量检索）
- 知识图谱增强检索（GraphRAG）
- 重排序（Rerank）

**目录结构：**
```
internal/
├── agent/              # Agent引擎
├── application/
│   ├── repository/    # 数据访问层
│   └── service/       # 业务逻辑层
├── handler/           # HTTP处理器
├── middleware/        # 中间件
├── models/            # 数据模型
├── router/            # 路由配置
├── config/            # 配置管理
├── database/          # 数据库初始化
├── logger/            # 日志
├── mcp/               # MCP客户端
└── searchutil/        # 检索工具
```

**启动流程：**
1. 加载配置文件 (`config/config.yaml`)
2. 初始化依赖注入容器
3. 连接数据库（PostgreSQL）
4. 执行数据库迁移
5. 初始化Redis客户端
6. 初始化文件存储服务
7. 初始化检索引擎
8. 启动HTTP服务器

### 3. 文档解析服务 (DocReader)

**技术栈：**
- Python 3.10+
- gRPC
- PaddleOCR (OCR)
- pdfplumber, python-docx (文档解析)
- Playwright (网页解析)

**主要功能：**
- PDF文档解析
- Word文档解析
- Markdown解析
- 图片OCR识别
- 图片Caption生成（多模态）
- 网页内容提取

**支持的文档格式：**
- PDF (.pdf)
- Word (.docx, .doc)
- Markdown (.md)
- 文本 (.txt)
- Excel (.xlsx, .xls)
- CSV (.csv)
- HTML (.html)
- 图片 (.png, .jpg, .jpeg)

**目录结构：**
```
docreader/
├── parser/            # 文档解析器
│   ├── pdf_parser.py
│   ├── docx_parser.py
│   ├── ocr_engine.py
│   └── ...
├── splitter/          # 文档分割器
├── models/            # 数据模型
├── proto/             # gRPC定义
└── main.py            # 服务入口
```

**gRPC接口：**
- `ParseDocument`: 解析文档
- `ParseImage`: 解析图片（OCR/Caption）
- `HealthCheck`: 健康检查

---

### 4. 数据存储层

#### 4.1 PostgreSQL (ParadeDB)
**用途：**
- 主数据库（用户、知识库、会话等）
- 向量存储（pgvector扩展）
- 全文检索（ParadeDB内置）

**主要表结构：**
- `tenants`: 租户表
- `knowledge_bases`: 知识库表
- `knowledges`: 知识条目表
- `chunks`: 文档块表（含向量）
- `sessions`: 会话表
- `messages`: 消息表
- `models`: 模型配置表
- `users`: 用户表

**向量索引：**
- 使用pgvector扩展存储embedding向量
- 支持余弦相似度、L2距离等检索

#### 4.2 Redis
**用途：**
- 缓存（会话上下文、检索结果等）
- 任务队列（Asynq）
- 分布式锁

**数据结构：**
- String: 缓存数据
- List: 任务队列
- Set: 分布式锁

#### 4.3 MinIO / 腾讯云COS
**用途：**
- 文档文件存储
- 图片存储
- 解析后的文件存储

**存储结构：**
```
bucket/
├── files/              # 原始文件
├── parsed/             # 解析后的文件
└── images/             # 图片文件
```

#### 4.4 Neo4j (可选)
**用途：**
- 知识图谱存储
- 实体关系图
- GraphRAG检索

**节点类型：**
- Entity: 实体节点
- Document: 文档节点
- Chunk: 文档块节点

**关系类型：**
- RELATED_TO: 实体关系
- CONTAINS: 文档包含关系
- SIMILAR_TO: 相似度关系

### 5. 外部服务集成

#### 5.1 大语言模型 (LLM)
**支持的模型：**
- Ollama本地模型（Qwen、DeepSeek等）
- OpenAI API
- 自定义API（兼容OpenAI格式）

**调用方式：**
- OpenAI兼容API
- 流式响应支持
- 思考模式/非思考模式

#### 5.2 嵌入模型 (Embedding)
**支持的模型：**
- BGE系列（BGE-large-zh-v1.5等）
- GTE系列
- OpenAI text-embedding-ada-002
- 本地模型（通过Ollama）

**向量维度：**
- BGE-large-zh-v1.5: 1024维
- text-embedding-ada-002: 1536维
- 可配置

#### 5.3 重排序模型 (Rerank)
**支持的模型：**
- BGE Reranker
- 自定义Rerank API

#### 5.4 MCP工具
**用途：**
- 扩展Agent能力
- 集成外部工具
- 支持多种传输方式（Stdio、HTTP、SSE）

**启动方式：**
- uvx启动器
- npx启动器
- 自定义启动器

## 🔄 数据流程

### 文档导入流程

```
1. 用户上传文档
   ↓
2. 前端 → 后端API (POST /api/v1/knowledge-bases/{id}/knowledges)
   ↓
3. 后端保存文件到存储（MinIO/COS/本地）
   ↓
4. 后端调用DocReader服务 (gRPC ParseDocument)
   ↓
5. DocReader解析文档
   ├── PDF → pdfplumber提取文本
   ├── Word → python-docx提取
   ├── 图片 → PaddleOCR识别
   └── 网页 → Playwright渲染提取
   ↓
6. DocReader返回解析结果
   ↓
7. 后端分割文档为Chunks
   ↓
8. 后端调用Embedding模型生成向量
   ↓
9. 后端存储Chunks和向量到PostgreSQL
   ↓
10. 可选：构建知识图谱（Neo4j）
```

### 问答流程

```
1. 用户提问
   ↓
2. 前端 → 后端API (POST /api/v1/sessions/{id}/messages)
   ↓
3. 后端查询改写（可选）
   ↓
4. 检索阶段
   ├── 关键词检索（BM25）
   ├── 向量检索（pgvector）
   └── 知识图谱检索（Neo4j，可选）
   ↓
5. 重排序（可选）
   ↓
6. 构建上下文
   ↓
7. 调用LLM生成回答
   ├── Agent模式：多次迭代，调用工具
   └── 普通模式：直接生成
   ↓
8. 流式返回结果
   ↓
9. 保存会话历史
```

### Agent模式流程

```
1. 用户提问（Agent模式）
   ↓
2. LLM思考（ReACT）
   ↓
3. 决定调用工具
   ├── 知识库检索工具
   ├── MCP工具
   └── Web搜索工具
   ↓
4. 执行工具调用
   ↓
5. 获取工具结果
   ↓
6. LLM反思和总结
   ↓
7. 判断是否需要继续迭代
   ├── 是 → 返回步骤2
   └── 否 → 生成最终答案
   ↓
8. 返回结果给用户
```

## 🔌 API设计

### RESTful API

**基础路径：** `/api/v1`

**主要端点：**

#### 知识库管理
- `GET /knowledge-bases` - 获取知识库列表
- `POST /knowledge-bases` - 创建知识库
- `GET /knowledge-bases/{id}` - 获取知识库详情
- `PUT /knowledge-bases/{id}` - 更新知识库
- `DELETE /knowledge-bases/{id}` - 删除知识库

#### 知识条目管理
- `POST /knowledge-bases/{id}/knowledges` - 上传文档
- `GET /knowledge-bases/{id}/knowledges` - 获取知识列表
- `DELETE /knowledges/{id}` - 删除知识

#### 会话管理
- `POST /sessions` - 创建会话
- `GET /sessions` - 获取会话列表
- `GET /sessions/{id}` - 获取会话详情
- `DELETE /sessions/{id}` - 删除会话

#### 消息管理
- `POST /sessions/{id}/messages` - 发送消息（流式）
- `GET /sessions/{id}/messages` - 获取消息历史

#### 模型管理
- `GET /models` - 获取模型列表
- `POST /models` - 添加模型
- `PUT /models/{id}` - 更新模型
- `DELETE /models/{id}` - 删除模型

**API文档：**
- Swagger UI: http://localhost:8080/swagger/index.html
- OpenAPI规范: `docs/swagger.yaml`
