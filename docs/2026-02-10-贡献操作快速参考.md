# 🚀 WeKnora 贡献操作快速参考

> **文档创建**：2026-02-10

## 首次设置（只需一次）

```bash
# 1. Fork 项目到你的 GitHub 账户（在 GitHub 网页上操作）

# 2. 克隆你的 Fork
git clone https://github.com/你的用户名/WeKnora.git
cd WeKnora

# 3. 添加上游仓库
git remote add upstream https://github.com/Tencent/WeKnora.git

# 4. 验证配置
git remote -v
```

## 什么是 Pull Request（提 PR）

**Pull Request（PR）** 是 GitHub 上的一种协作方式：你**请求对方仓库“拉取”你 Fork 里的某条分支的改动**，把你这边的代码“合并进”上游项目的正式代码里。

| 概念 | 说明 |
|------|------|
| **谁在“提”** | 你（贡献者），在**自己的 Fork**（如 `sunlingjun/WeKnora`）里开发并推送分支 |
| **提给谁** | **上游仓库**（如 `Tencent/WeKnora`）的维护者 |
| **提的是什么** | 一个“请求”：请把我在 Fork 上的 **某条分支**（如 `feature/xxx`）的提交，**合并到上游的 main** |
| **“拉取”的含义** | 上游维护者从你的 Fork 拉取（pull）你这边的改动，审阅后合并进他们的分支 |

**流程简图：**

```
你的电脑（本地）  →  git push  →  你的 Fork（GitHub）
                                      ↓
                             在网页上点「New Pull Request」
                                      ↓
Tencent/WeKnora（上游）  ←  维护者审阅并 Merge  ←  你选的分支（如 feature/xxx）
```

**“提 PR”在网页上具体做什么：**

1. 打开 **你的 Fork** 页面：`https://github.com/sunlingjun/WeKnora`
2. 选分支为刚推送的 `feature/xxx`，点 **「Compare & pull request」**（或菜单里的 Pull requests → New）
3. **Base 仓库**选上游（如 `Tencent/WeKnora`）的 `main`，**compare 分支**选你 Fork 的 `feature/xxx`
4. 写清楚**标题**和**描述**（改了什么、为什么改、如何验证），点 **「Create pull request」**

提交后，上游维护者会在该 PR 里看代码、提意见；你根据反馈改代码并 `git push origin feature/xxx`，他们确认无误后会点 **Merge**，你的改动就正式进入上游仓库。  
**“提 PR”= 在 GitHub 上创建并提交这样一个合并请求。**

## 日常贡献流程

### 1️⃣ 同步最新代码

```bash
git checkout main
git fetch upstream
git merge upstream/main
git push origin main
```

### 2️⃣ 创建功能分支

```bash
git checkout -b feature/功能名称
# 或
git checkout -b fix/问题描述
```

### 3️⃣ 进行开发

```bash
# 后端开发
make dev-start
make dev-app

# 前端开发
cd frontend
npm run dev
```

### 4️⃣ 提交更改

```bash
# 查看更改
git status
git diff

# 暂存更改
git add .

# 提交（使用 Conventional Commits 格式）
git commit -m "feat: 添加新功能"

# 推送分支
git push origin feature/功能名称
```

### 5️⃣ 创建 Pull Request

1. 访问 `https://github.com/你的用户名/WeKnora`
2. 点击 "Compare & pull request"
3. 填写 PR 标题和描述
4. 点击 "Create pull request"

### 6️⃣ 处理审查反馈

```bash
# 修改代码后
git add .
git commit -m "fix: 修复审查反馈的问题"
git push origin feature/功能名称
```

### 7️⃣ PR 合并后清理

```bash
git checkout main
git pull upstream main
git branch -d feature/功能名称
```

## 常用 Git 命令

```bash
# 查看当前状态
git status

# 查看更改内容
git diff

# 查看提交历史
git log --oneline

# 撤销未暂存的更改
git checkout -- 文件名

# 撤销已暂存但未提交的更改
git reset HEAD 文件名

# 修改最后一次提交
git commit --amend -m "新的提交信息"

# 查看远程分支
git branch -r

# 查看所有分支
git branch -a
```

## 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型**:
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档
- `style`: 格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例**:
```bash
git commit -m "feat(frontend): 添加颜色统一化功能"
git commit -m "fix(api): 修复向量检索精度问题"
git commit -m "docs: 更新贡献指南"
```

## 分支命名规范

- `feature/功能名称` - 新功能
- `fix/问题描述` - Bug 修复
- `docs/文档主题` - 文档改进
- `refactor/重构内容` - 代码重构
- `test/测试内容` - 测试相关
