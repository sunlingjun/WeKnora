## 详细合并说明

### 1. theme.css（完全保留本地）

```powershell
# 这是CSS变量主题系统，必须完全保留本地版本
git checkout HEAD -- frontend/src/assets/theme/theme.css
git add frontend/src/assets/theme/theme.css
```

### 2. router.go（合并双方路由）

需要确保以下路由同时存在：

```go
// 远程新增路由
r.GET("/agents/:id/suggested-questions", agentHandler.GetSuggestedQuestions)

// 本地CAS路由（必须保留）
r.POST("/cas/validate", casHandler.Validate)
r.POST("/cas/logout", casHandler.Logout)
```

### 3. main.go（接受远程改进）

远程新增内容：
- `listenWithRetry` 函数（端口重试监听）
- 双信号处理（优雅关闭）
- 资源清理改进

**检查点**：
- CAS客户端初始化未被覆盖
- 资源清理器包含所有组件

### 4. knowledge.go（合并Channel参数）

保留双方的修改：

```go
// 1. 保留远程的Channel参数
func CreateKnowledgeFromFile(..., channel string) (*Knowledge, error)

// 2. 保留本地可能有的共享知识库扩展
```

## 紧急回滚

```powershell
# 方案1: 中止合并
git merge --abort

# 方案2: 回滚到备份
git reset --hard backup/merge-before-YYYYMMDD

# 方案3: 使用补丁恢复
git apply patches/cas_auth.patch
git apply patches/theme.patch
```

## 常见问题

### Q: 合并后CAS不工作？

```powershell
# 检查CAS文件是否被覆盖
git diff HEAD -- internal/handler/cas_auth.go

# 恢复CAS文件
git checkout HEAD -- internal/handler/cas_auth.go
git checkout HEAD -- internal/application/service/cas_auth.go
git checkout HEAD -- internal/application/service/cas_client.go
git checkout HEAD -- frontend/src/stores/cas.ts
```

### Q: 合并后颜色显示异常？

```powershell
# 检查theme.css是否完整
git diff HEAD -- frontend/src/assets/theme/theme.css

# 恢复theme.css
git checkout HEAD -- frontend/src/assets/theme/theme.css
```

### Q: 合并后知识库列表异常？

```powershell
# 检查共享知识库服务
git diff HEAD -- internal/application/service/shared_kb.go

# 检查前端组件
git diff HEAD -- frontend/src/views/knowledge/SharedKnowledgeBaseSquare.vue
```
