## 总体执行顺序

**建议执行顺序**：
1. **先完成 CAS 单点登录集成**（功能1）
2. **再完成共享知识库功能**（功能2）

**原因**：
- CAS 登录是基础功能，用户认证后才能使用知识库功能
- 共享知识库功能依赖用户体系，需要先完成用户认证

## 功能二：共享知识库功能

### 阶段一：数据库迁移

#### 步骤 1.1：创建知识库表扩展迁移文件

**文件路径**：`migrations/versioned/000013_add_shared_kb_fields.up.sql`

**文件内容**：
```sql
-- 添加共享知识库相关字段
ALTER TABLE knowledge_bases ADD COLUMN IF NOT EXISTS visibility VARCHAR(32) DEFAULT 'private';  -- 'private' | 'shared'
ALTER TABLE knowledge_bases ADD COLUMN IF NOT EXISTS owner_id VARCHAR(36);  -- 创建者用户 ID（关联 users.id）
ALTER TABLE knowledge_bases ADD COLUMN IF NOT EXISTS shared_at TIMESTAMP WITH TIME ZONE;  -- 共享时间
ALTER TABLE knowledge_bases ADD COLUMN IF NOT EXISTS member_count INTEGER DEFAULT 0;  -- 成员数量（冗余字段）

-- 添加索引
CREATE INDEX IF NOT EXISTS idx_knowledge_bases_visibility ON knowledge_bases(visibility);
CREATE INDEX IF NOT EXISTS idx_knowledge_bases_owner_id ON knowledge_bases(owner_id);
CREATE INDEX IF NOT EXISTS idx_knowledge_bases_shared_at ON knowledge_bases(shared_at);
```

**验证点**：
- [ ] 迁移文件语法正确
- [ ] 字段类型和默认值正确
- [ ] 索引创建正确

#### 步骤 1.2：创建成员表迁移文件

**文件路径**：`migrations/versioned/000014_create_kb_members.up.sql`

**文件内容**：
```sql
-- 共享知识库成员表
CREATE TABLE IF NOT EXISTS knowledge_base_members (
    id VARCHAR(36) PRIMARY KEY DEFAULT uuid_generate_v4(),
    knowledge_base_id VARCHAR(36) NOT NULL,
    user_id VARCHAR(36) NOT NULL,
    tenant_id INTEGER NOT NULL,
    role VARCHAR(32) NOT NULL DEFAULT 'viewer',  -- 'owner' | 'editor' | 'viewer'
    joined_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    deleted_at TIMESTAMP WITH TIME ZONE,
    
    CONSTRAINT fk_kb_members_kb FOREIGN KEY (knowledge_base_id) REFERENCES knowledge_bases(id) ON DELETE CASCADE,
    CONSTRAINT fk_kb_members_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    CONSTRAINT fk_kb_members_tenant FOREIGN KEY (tenant_id) REFERENCES tenants(id) ON DELETE CASCADE,
    CONSTRAINT uk_kb_members_kb_user UNIQUE (knowledge_base_id, user_id, deleted_at)
);

-- 添加索引
CREATE INDEX IF NOT EXISTS idx_kb_members_kb_id ON knowledge_base_members(knowledge_base_id);
CREATE INDEX IF NOT EXISTS idx_kb_members_user_id ON knowledge_base_members(user_id);
CREATE INDEX IF NOT EXISTS idx_kb_members_tenant_id ON knowledge_base_members(tenant_id);
CREATE INDEX IF NOT EXISTS idx_kb_members_role ON knowledge_base_members(role);
```

**验证点**：
- [ ] 表结构定义正确
- [ ] 外键约束正确
- [ ] 唯一约束正确（支持软删除）
- [ ] 索引创建正确

## 注意事项

### 开发顺序建议

1. **先完成 CAS 单点登录集成**
   - 这是基础功能，其他功能依赖用户认证
   - 完成后可以验证用户登录和认证流程

2. **再完成共享知识库功能**
   - 依赖用户体系，需要先完成 CAS 登录
   - 可以逐步实现：先实现基础功能，再完善权限管理

### 关键验证点

1. **CAS 登录**：
   - 环境切换（生产/测试）是否正确
   - Cookie 读取是否正确（根据环境）
   - 用户绑定逻辑是否正确（查找优先级）
   - 密码生成是否正确（手机号后四位）

2. **共享知识库**：
   - 权限检查是否正确
   - 成员管理是否正确
   - 对话模块知识库选择是否正确
   - 数据一致性（成员数量更新）

### 测试重点

1. **CAS 登录**：
   - 不同环境的 Cookie 读取
   - 用户自动绑定和创建
   - 租户自动创建和绑定
   - JWT Token 生成和验证

2. **共享知识库**：
   - 创建和加入流程
   - 权限检查（read/write/delete）
   - 成员管理（角色修改、移除）
   - 对话模块知识库选择
