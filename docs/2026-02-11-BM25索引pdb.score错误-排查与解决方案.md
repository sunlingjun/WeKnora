# BM25 索引 pdb.score 错误：排查与解决方案

> **文档创建**：2026-02-11  
> 数据库升级后 ParadeDB BM25 索引导致 UPDATE/DELETE 报错的完整梳理

---

## 一、问题描述

### 1.1 错误信息

```
ERROR: function "pdb.score(anyelement)" does not exist (SQLSTATE 42883)
```

**触发场景**：
- 删除知识库时（级联删除 embeddings）
- 执行 `UPDATE embeddings SET is_enabled = false WHERE ...`
- 执行 `DELETE FROM embeddings WHERE ...`

### 1.2 根本原因

ParadeDB BM25 索引（`embeddings_search_idx`）在 **INSERT / UPDATE / DELETE** 时会触发内部索引重算，并调用 `pdb.score()` 函数。在部分场景下该调用失败，导致报错。

---

## 二、BM25 索引简介

| 项目 | 说明 |
|------|------|
| 索引名 | `embeddings_search_idx` |
| 类型 | ParadeDB BM25 全文搜索 |
| 分词器 | `chinese_lindera` |
| 用途 | 关键词搜索、混合检索中的 BM25 部分 |

**使用位置**：`internal/application/repository/retriever/postgres/repository.go`  
- `KeywordsRetrieve()`：`paradedb.match()`、`paradedb.score(id)`

---

## 三、尝试过的方案及结论

| 方案 | 做法 | 结果 |
|------|------|------|
| 1. 部分索引 | `WHERE (is_enabled IS NULL OR is_enabled = true)` | 可减轻，但 UPDATE/DELETE 仍可能触发错误 |
| 2. UPDATE 加条件 | `AND (is_enabled IS NULL OR is_enabled = true)` | 仍可能报错 |
| 3. 跳过 UPDATE，直接 DELETE | 不执行 UPDATE，只执行 DELETE | DELETE 本身也会触发错误，无效 |
| 4. **临时删除索引** | 删除索引 → 执行操作 → 重建索引 | ✅ 可行 |

---

## 四、最终解决方案：临时删除索引

### 4.1 思路

在执行 DELETE/UPDATE 前先删除 BM25 索引，操作完成后再重建，从而避免索引更新时调用 `pdb.score()`。

### 4.2 代码实现要点

**文件**：`internal/application/repository/retriever/postgres/repository.go`

```go
// deleteByColumn 删除流程
func (g *pgRepository) deleteByColumn(...) error {
    // Step 1: 临时删除 BM25 索引
    g.db.Exec("DROP INDEX IF EXISTS embeddings_search_idx")
    
    // Step 2: 执行删除
    g.db.Exec("DELETE FROM embeddings WHERE ...")
    
    // Step 3: 重建 BM25 部分索引
    g.db.Exec(`CREATE INDEX embeddings_search_idx ON embeddings 
        USING bm25 (id, knowledge_base_id, content, knowledge_id, chunk_id)
        WITH (key_field = 'id', text_fields = '{"content": {"tokenizer": {"type": "chinese_lindera"}}}')
        WHERE (is_enabled IS NULL OR is_enabled = true)`)
}
```

### 4.3 优缺点

| 优点 | 缺点 |
|------|------|
| 避免 `pdb.score` 报错 | 删除/重建索引期间 BM25 搜索短暂不可用 |
| 保留 BM25 搜索能力 | 删除与重建有一定耗时 |
| 逻辑简单、可靠 | 需注意并发与错误处理 |

---

## 五、删除 BM25 索引的影响

### 5.1 不受影响

- 向量搜索（HNSW 索引）
- 数据插入
- 若永久删除 BM25，则 UPDATE/DELETE 不再报错

### 5.2 受影响

- 关键词搜索：依赖 `paradedb.match()` 和 BM25 索引
- 混合搜索：BM25 部分失效

### 5.3 方案选择

| 场景 | 建议 |
|------|------|
| 需要 BM25 / 关键词搜索 | 采用**临时删除索引**方案（当前实现） |
| 不需要 BM25 | 可**永久删除** BM25 索引，并禁用关键词搜索 |

---

## 六、SQL 脚本

### 6.1 永久删除 BM25 索引（不再使用关键词搜索时）

```sql
DROP INDEX IF EXISTS embeddings_search_idx;
```

### 6.2 重建 BM25 部分索引

```sql
CREATE INDEX embeddings_search_idx ON embeddings
USING bm25 (id, knowledge_base_id, content, knowledge_id, chunk_id)
WITH (
    key_field = 'id',
    text_fields = '{"content": {"tokenizer": {"type": "chinese_lindera"}}}'
)
WHERE (is_enabled IS NULL OR is_enabled = true);
```

### 6.3 验证索引

```sql
SELECT indexname, indexdef 
FROM pg_indexes 
WHERE tablename = 'embeddings' AND indexname = 'embeddings_search_idx';
```

## 八、总结

- **问题**：ParadeDB BM25 索引在 UPDATE/DELETE 时触发 `pdb.score()` 导致报错。
- **推荐方案**：在执行相关操作前临时删除 BM25 索引，操作完成后重建。
- **备选**：若不需要关键词搜索，可永久删除 BM25 索引并禁用相关功能。
